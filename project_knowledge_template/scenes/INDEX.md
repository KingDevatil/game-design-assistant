# 自定义场景索引

> 用户可在项目知识包中定义**专属场景**，扩展 Skill 的能力覆盖。
> 自定义场景遵循与内置场景相同的结构规范，优先级高于同ID的内置场景。

## 场景清单

| 场景ID | 名称 | 关联角色 | 触发关键词 | 描述 |
|--------|------|----------|-----------|------|
| `{scene_id}` | {场景名称} | `{role_id}` | {关键词1}、{关键词2} | {场景用途} |

> **说明**：`关联角色` 填写角色ID（如 `system-designer`、`writer`、`combat-designer` 等），表示该场景属于哪个角色的工作范畴。

## 使用规则

1. **文件命名**：每个场景一个独立文件，文件名建议与场景ID一致（如 `hero_design.md`）
2. **场景ID**：建议使用有意义的英文标识，如 `CBT-boss_design`（战斗策划-Boss设计）
3. **文件格式**：遵循与内置场景相同的结构规范（见下方模板），包含：
   - 场景标识（ID / 名称 / 岗位 / 优先级）
   - 场景目的
   - 输入模板（用户需要提供的信息）
   - 处理逻辑（AI 的处理步骤）
   - 输出规范（AI 的输出模板）
   - 质量自检清单
4. **优先级**：**自定义场景 > 内置场景** — 当自定义场景的 ID 与内置场景重复时，以自定义场景为准（P0 级覆盖）
