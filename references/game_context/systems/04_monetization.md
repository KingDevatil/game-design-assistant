# 商业化系统

> **定义**：将玩家行为转化为收入，是手游的生命线。设计合理的付费点与转化路径。
> **适用范围**：所有含付费设计的游戏

---

## 1. 抽卡/祈愿系统

### 1.1 卡池设计
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **角色池** | 抽取角色/英雄 | 核心付费点 |
| **装备池** | 抽取武器/装备 | 次级付费点 |
| **常驻池** | 长期存在的标准卡池 | 新手引导期使用 |
| **限定池** | 限时UP特定角色 | 制造紧迫感 |
| **友情池** | 消耗友情点抽取 | 免费玩家福利 |

### 1.2 概率设计
| 要素 | 说明 | 典型数值 |
|---|---|---|
| **基础概率** | SSR/5星基础出率 | 0.6%-2% |
| **综合概率** | 含保底的平均出率 | 1.5%-3% |
| **软保底** | 一定次数后概率提升 | 70-90次 |
| **硬保底** | 达到次数必出 | 90-100次 |
| **大小保底** | 第一次保底50%UP，第二次必UP | 防止无限歪 |

### 1.3 卡池节奏
| 周期 | 设计要点 |
|---|---|
| **双周卡池** | 每两周更换UP角色 |
| **版本卡池** | 大版本更新配合新角色 |
| **复刻卡池** | 老角色返场 |
| **节日卡池** | 春节/周年庆等特殊卡池 |

**典型案例**：《原神》祈愿系统、《明日方舟》寻访系统、《崩坏：星穹铁道》跃迁系统

## 2. 通行证/战令系统

### 2.1 通行证结构
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **免费档** | 所有玩家可领取的基础奖励 | 保证基础体验 |
| **付费档** | 购买后解锁的高级奖励 | 价值远超价格 |
| **等级设计** | 通常50-100级 | 每级奖励价值递增 |
| **经验获取** | 完成任务获得经验升级 | 任务类型多样化 |

### 2.2 通行证设计要点
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **价值感知** | 付费档总价值是价格的10-20倍 | 让玩家觉得"超值" |
| **完成率** | 设计合理的任务量，保证50%+玩家能满级 | 避免挫败感 |
| **时间压力** | 赛季制，到期后无法获取 | 制造紧迫感 |
| **付费直升** | 可付费购买等级 | 为大R提供捷径 |

**典型案例**：《王者荣耀》战令、《原神》纪行、《和平精英》赛季手册

## 3. 商城系统

### 3.1 商城类型
| 类型 | 说明 | 设计要点 |
|---|---|---|
| **基础商城** | 出售常规道具、材料 | 价格稳定，满足日常需求 |
| **神秘/黑市商城** | 随机刷出稀有道具，折扣出售 | 创造"捡漏"惊喜感 |
| **竞技场商城** | 用竞技币兑换 | PVP专属奖励 |
| **公会商城** | 用公会币兑换 | 鼓励公会活跃 |
| **活动商城** | 限时开放，活动货币兑换 | 活动专属道具 |
| **充值商城** | 直接充值购买 | 首充双倍、充值档位 |

### 3.2 礼包设计
| 类型 | 说明 | 定价策略 |
|---|---|---|
| **月卡** | 每日领取奖励，持续30天 | 高性价比，培养付费习惯，通常30元 |
| **首充礼包** | 首次充值赠送额外奖励 | 极低价（1元/6元）吸引首次付费 |
| **限时礼包** | 弹窗/活动期间的限时优惠 | 倒计时+限量，制造紧迫感 |
| **成长基金** | 达到指定等级返还多倍钻石 | 前期投资，后期回报 |
| **周卡/终身卡** | 长期收益型礼包 | 锁定长期留存 |

### 3.3 定价策略
| 原则 | 说明 |
|---|---|
| **锚定效应** | 设置高价礼包作为锚点，让中档显得划算 |
| **价格梯度** | 6/30/68/128/328/648等档位 |
| **首充双倍** | 首次充值双倍钻石，降低首次付费门槛 |
| **累充奖励** | 累计充值达到档位获得额外奖励 |

## 4. 首充/续充系统

### 4.1 首充设计
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **首充奖励** | 首次任意金额充值获得 | 奖励价值要高 |
| **首充档位** | 通常1元/6元档位 | 极低门槛 |
| **首充内容** | SSR角色/稀有皮肤/大量资源 | 让玩家觉得"血赚" |

### 4.2 续充设计
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **续充奖励** | 持续充值获得额外奖励 | 培养付费习惯 |
| **累充档位** | 累计充值达到档位获得奖励 | 档位设计要有吸引力 |
| **VIP系统** | 累计充值提升VIP等级 | VIP特权设计 |

## 5. 广告变现系统

### 5.1 广告类型
| 类型 | 说明 | 设计要点 |
|---|---|---|
| **激励视频** | 观看广告获得奖励 | 奖励要有吸引力 |
| **插屏广告** | 自然断点展示的全屏广告 | 频率控制 |
| **Banner广告** | 底部横幅广告 | 对体验影响较小 |
| **开屏广告** | 启动时展示 | 可接受 |

### 5.2 广告设计约束
- **频率控制**：避免过度广告引起反感
- **奖励平衡**：广告奖励要有价值但不能破坏经济
- **用户选择**：广告应是可选的，不能强制

## 6. 付费转化漏斗

### 6.1 转化路径
```
免费玩家 → 首充玩家 → 小额付费 → 中额付费 → 大额付费（鲸鱼用户）
```

### 6.2 各阶段策略
| 阶段 | 策略 | 目标 |
|---|---|---|
| **免费→首充** | 首充礼包、限时优惠 | 3-8%转化率 |
| **首充→小额** | 月卡、战令 | 培养付费习惯 |
| **小额→中额** | 限定卡池、限时礼包 | 提升ARPU |
| **中额→大额** | 累充活动、排行榜 | 挖掘鲸鱼用户 |

---

## 7. 设计检查清单

- [ ] 付费点是否清晰？玩家是否知道钱花在哪里？
- [ ] 首充转化是否顺畅？首充奖励是否有吸引力？
- [ ] 抽卡体验是否健康？保底机制是否完善？
- [ ] 通行证完成率是否合理？任务量是否适中？
- [ ] 付费是否破坏游戏平衡？（尤其竞技类游戏）
- [ ] 付费提示是否适度？是否存在过度弹窗？
- [ ] 是否有明确的付费天花板？
