# 核心玩法系统

> **定义**：游戏"玩什么"，是玩家最核心的交互体验，也是整个游戏的乐趣入口。
> **适用范围**：所有游戏

---

## 1. 战斗系统

### 1.1 回合制战斗
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **行动顺序** | 按速度/敏捷决定出手顺序 | 速度差要明显，避免乱速 |
| **技能释放** | 消耗MP/怒气释放技能 | 技能要有明显差异化 |
| **属性克制** | 五行/元素相克关系 | 克制伤害加成20-50% |
| **阵型/站位** | 前后排、职业站位影响 | 增加策略深度 |

**典型案例**：《阴阳师》、《梦幻西游》、《崩坏：星穹铁道》

### 1.2 ARPG即时战斗
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **动作连招** | 普攻与技能可衔接 | 操作上限和爽感的来源 |
| **闪避/格挡** | 防御性操作，有无敌帧 | 时机把握是技术核心 |
| **元素反应** | 不同元素组合产生特效 | 增加Build多样性 |
| **视角控制** | 自由视角或锁定视角 | 视角舒适度影响体验 |

**典型案例**：《原神》、《崩坏3》、《战双帕弥什》

### 1.3 塔防战斗
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **防御塔类型** | 输出/控制/辅助/阻挡 | 塔间要有配合 |
| **敌人路线** | 固定路线或自由布置 | 路线设计影响策略 |
| **资源管理** | 击杀敌人获得费用部署 | 前期紧张后期爽快 |
| **英雄单位** | 可移动的英雄角色 | 增加操作维度 |

**典型案例**：《明日方舟》、《王国保卫战》、《植物大战僵尸》

### 1.4 伤害公式设计
```
基础伤害 = 攻击力 × 技能倍率
实际伤害 = 基础伤害 × (1 + 增伤%) × (1 - 防御减免%) × 暴击倍数 × 随机波动
```

| 参数 | 说明 | 典型范围 |
|---|---|---|
| **防御减免** | 防御/(防御+常数) 或 百分比减伤 | 常数为固定数值（如1000），或设为防御力×系数 |
| **暴击率** | 触发暴击的概率 | 基础5-20%，可堆至50-80% |
| **暴击伤害** | 暴击时的伤害倍数 | 通常150-200% |
| **随机波动** | 伤害浮动范围 | 通常±5-10% |

## 2. 关卡/副本系统

### 2.1 主线关卡
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **章节结构** | 按剧情分章节，每章若干关卡 | 章节长度适中（8-15关） |
| **难度曲线** | 平缓上升，Boss关有明显难度提升 | 避免卡关劝退 |
| **三星评价** | 通关条件拆分（通关/限时/无伤等） | 增加重复挑战动力 |
| **扫荡功能** | 三星通关后可快速消耗体力获取奖励 | 减负必备 |

### 2.2 资源副本
| 类型 | 产出 | 设计要点 |
|---|---|---|
| **经验副本** | 角色经验道具 | 每日次数限制，产出稳定 |
| **金币副本** | 基础货币 | 同上 |
| **材料副本** | 突破/技能升级材料 | 分不同材料分日开放 |
| **装备副本** | 装备或装备材料 | 分部位/品质分日开放 |

### 2.3 爬塔/深渊
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **层数设计** | 每层难度递增 | 每10层一个Boss，每50层大奖励 |
| **进度保存** | 失败后可从当前层重试 | 避免重复爬低层 |
| **每日重置** | 已通关层数可扫荡 | 减负设计 |
| **buff选择** | 部分爬塔提供随机buff | 增加策略性 |

**典型案例**：《原神》深境螺旋、《明日方舟》危机合约

### 2.4 Roguelike玩法
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **局内Build** | 每局从基础开始，通过选择成长 | 与局外养成解耦 |
| **路线选择** | 分支路线选择战斗/商店/事件 | 玩家自主决策 |
| **buff/遗物** | 随机获取的增益效果 | buff间要有联动（Synergy） |
| **局外解锁** | 通过游玩解锁新内容 | 降低挫败感 |

## 3. 探索系统

### 3.1 大世界探索
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **地图密度** | 兴趣点分布密度 | 既不能太空也不能太满 |
| **奖励分布** | 宝箱、收集品散落各处 | 引导玩家探索 |
| **引导节奏** | 显性与隐性引导结合 | 避免玩家迷失 |
| **快速移动** | 传送点、坐骑等 | 减少跑图疲劳 |

**典型案例**：《原神》、《塞尔达传说》、《巫师3》

### 3.2 解谜玩法
| 类型 | 说明 | 设计要点 |
|---|---|---|
| **环境解谜** | 开关、推箱子、光线折射 | 难度适中，提示友好 |
| **逻辑解谜** | 数独、华容道、密码 | 提供思考乐趣 |
| **剧情解谜** | 根据线索推理真相 | 增强叙事沉浸感 |

## 4. PvP竞技系统

### 4.1 异步PVP（竞技场）
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **防守阵容** | 玩家设置防守队伍 | 可与进攻阵容不同 |
| **排名机制** | 击败高排名互换排名 | 越往上对手越强 |
| **每日奖励** | 根据排名发放奖励 | 鼓励冲击高排名 |
| **赛季结算** | 定期重置，发放赛季奖励 | 保持新鲜感 |

### 4.2 实时PVP
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **匹配机制** | 基于战力/段位匹配 | 匹配时间控制在30秒内 |
| **段位系统** | 青铜→白银→黄金→铂金→钻石→王者 | 有段位保护机制 |
| **Ban/Pick** | 禁用/选择角色 | 增加策略深度 |
| **延迟补偿** | 网络延迟处理 | 技术实现关键 |

### 4.3 公会战/阵营战
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **领地争夺** | 地图据点占领获取积分 | 战略选择和分工 |
| **个人贡献** | 伤害/占领/击杀贡献 | 奖励与贡献挂钩 |
| **匹配公平** | 实力相近的公会匹配 | 避免碾压 |

## 5. 设计检查清单

- [ ] 战斗节奏是否流畅？操作反馈是否及时？
- [ ] 难度曲线是否合理？是否存在突然的难度跳跃？
- [ ] 是否有足够的策略深度？Build多样性如何？
- [ ] PVP是否公平？付费对平衡性的影响？
- [ ] 关卡长度是否合适？是否存在疲劳点？
- [ ] 探索奖励是否足够吸引？隐藏要素是否有趣？
