# 经济系统

> **定义**：管理游戏内资源的产出与消耗，是数值策划的核心战场。背包作为资源的中转节点也归入此类。
> **适用范围**：所有含资源循环的游戏

---

## 1. 货币体系

### 1.1 货币分层
| 层级 | 货币名称 | 获取方式 | 主要用途 | 设计原则 |
|---|---|---|---|---|
| **基础货币** | 金币/铜钱/银币 | 挂机、副本、出售道具 | 强化、升级、日常消耗 | 大量产出，大量消耗，防止通胀 |
| **高级货币** | 钻石/元宝/水晶 | 充值、活动、成就 | 抽卡、购买稀有道具、加速 | 控制产出，核心付费锚点 |
| **竞技货币** | 竞技币/荣耀点 | PVP、竞技场 | 兑换PVP专属奖励 | 鼓励PVP参与 |
| **公会货币** | 贡献点/公会币 | 公会活动 | 公会商店兑换 | 鼓励公会活跃 |
| **活动货币** | 代币/礼券 | 限时活动 | 活动商店兑换 | 活动期有效，过期回收 |
| **体力/能量** | 体力/AP/疲劳值 | 时间恢复、购买、道具 | 限制副本/关卡进入次数 | 控制玩家进度，制造付费点 |

### 1.2 货币设计约束
- **种类控制**：核心货币种类不宜过多（5-7种为宜，含进度资源），避免玩家认知混乱。基础+高级+2-3种玩法货币+活动货币+体力为常见配置
- **汇率设计**：货币间转换要有合理汇率，避免套利
- **免费产出**：免费货币产出要足够支撑核心体验，但进度慢于付费玩家
- **活动货币回收**：活动货币到期回收要温和（可转为少量基础货币），避免玩家反感

## 2. 产出系统

### 2.1 产出途径
| 途径 | 说明 | 设计要点 |
|---|---|---|
| **关卡掉落** | 通关副本获得奖励 | 难度越高奖励越好 |
| **任务奖励** | 主线/支线/日常任务 | 引导玩家行为 |
| **活动产出** | 限时活动额外产出 | 制造参与动力 |
| **挂机收益** | 离线后按时间积累 | 有上限（如8/12/24小时） |
| **成就奖励** | 完成成就获得 | 一次性奖励 |

### 2.2 产出节奏
| 周期 | 设计要点 |
|---|---|
| **每小时** | 体力恢复、挂机收益 |
| **每日** | 日常任务、每日副本、签到 |
| **每周** | 周常任务、周常副本、周末双倍 |
| **每月** | 月卡刷新、月度活动 |
| **版本** | 版本更新奖励、新内容产出 |

### 2.3 产出控制
- **稀缺性控制**：核心资源要有明确的产出上限
- **期望值管理**：随机奖励要设置保底
- **产出监控**：定期监控各类资源的人均持有量

---

## 3. 消耗系统

### 3.1 消耗途径
| 途径 | 说明 | 设计要点 |
|---|---|---|
| **强化消耗** | 装备/角色强化消耗 | 随等级提升消耗递增 |
| **抽卡消耗** | 角色/装备抽卡 | 核心付费点 |
| **体力消耗** | 进入副本消耗体力 | 控制进度 |
| **复活消耗** | 战斗失败复活 | 轻度惩罚 |
| **修理消耗** | 装备耐久度修理 | 货币回收手段 |

### 3.2 消耗节奏
- **消耗口径**：设计明确的资源消耗速率
- **回收速度**：货币回收速度要与产出平衡
- **经济循环**：产出→消耗→再产出形成闭环

---

## 4. 背包系统

### 4.1 背包功能
| 功能 | 说明 | 设计要点 |
|---|---|---|
| **物品存储** | 存放各类道具、材料 | 分类清晰，便于查找 |
| **容量管理** | 背包格子数量限制 | 初始容量+扩展机制 |
| **扩容机制** | 消耗货币/道具扩展容量 | 适度商业化 |
| **整理功能** | 自动整理、一键出售 | 减负设计 |

### 4.2 背包设计约束
- **容量对经济的影响**：背包容量会影响资源囤积行为
- **扩容商业化**：背包扩展可作为付费点，但不能太逼氪
- **仓库系统**：长期存储与日常背包分离

---

## 5. 交易系统

### 5.1 交易类型
| 类型 | 说明 | 设计要点 |
|---|---|---|
| **自由交易** | 玩家间直接交易 | 适合开放经济，但易被工作室利用 |
| **拍卖行** | 玩家挂单，其他玩家购买 | 收取手续费（货币回收） |
| **一口价/竞拍** | 固定价格或拍卖竞价 | 稀有物品适合竞拍 |
| **交易税** | 成交后扣除税费 | 主要货币回收手段 |
| **公示期** | 高价值物品上架后有公示期 | 防止盗号转移资产 |

### 5.2 交易设计约束
- **绑定机制**：绑定道具不可交易，防止工作室
- **价格管控**：设置最高/最低价，或完全由市场决定
- **交易记录**：完整记录便于追溯异常

---

## 6. 经济健康监控

### 6.1 关键指标
| 指标 | 说明 | 健康范围 |
|---|---|---|
| **通胀率** | 货币购买力下降速度 | 月通胀率<10% |
| **货币产出/消耗比** | 玩家获取vs消耗的平衡 | 接近1:1，略产出差可接受 |
| **资源囤积** | 玩家持有货币/材料总量 | 过高说明消耗不足 |
| **交易活跃度** | 拍卖行成交频率 | 反映经济活力 |

### 6.2 调控手段
| 问题 | 调控手段 |
|---|---|
| **通货膨胀** | 增加消耗途径、减少产出、提高交易税 |
| **通货紧缩** | 增加产出、减少消耗、刺激消费 |
| **资源囤积** | 新增消耗途径、限时活动消耗 |
| **工作室泛滥** | 加强绑定机制、交易限制、封号打击 |

---

## 7. 设计检查清单

- [ ] 货币种类是否清晰？玩家是否能理解各货币用途？
- [ ] 产出与消耗是否平衡？是否存在通胀/通缩风险？
- [ ] 免费玩家是否能获得足够的资源？
- [ ] 付费点设计是否合理？是否存在逼氪感？
- [ ] 交易系统是否健康？工作室影响如何？
- [ ] 经济监控指标是否完善？是否有预警机制？
