# 运营活动系统

> **定义**：持续提供新鲜内容，维持DAU和收入节奏。让玩家有持续登录的理由。
> **适用范围**：所有长线运营的游戏

---

## 1. 日常/周常系统

### 1.1 每日任务
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **任务类型** | 登录、消耗体力、完成副本、PVP等 | 覆盖核心玩法 |
| **任务数量** | 通常5-10个 | 完成时间控制在30分钟内 |
| **奖励设计** | 基础资源+活跃积分 | 积分兑换额外奖励 |
| **一键领取** | 批量领取已完成任务 | 减负设计 |

### 1.2 每周任务
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **任务难度** | 比日常任务更具挑战 | 如"击败100只怪物" |
| **奖励价值** | 比日常更丰厚的奖励 | 鼓励周中积累 |
| **周常副本** | 每周特定时间开放 | 周末双倍等设计 |

### 1.3 签到系统
| 类型 | 说明 | 设计要点 |
|---|---|---|
| **每日签到** | 每日登录领取奖励 | 奖励递增，第7天最好 |
| **累计签到** | 月内累计签到达天数有额外奖励 | 允许漏签几天 |
| **补签机制** | 消耗少量货币补签 | 降低玩家焦虑 |
| **月签到卡** | 购买后每日签到奖励翻倍 | 促进付费+每日登录 |

## 2. 限时活动系统

### 2.1 活动类型
| 类型 | 说明 | 设计要点 |
|---|---|---|
| **角色UP池** | 特定角色抽卡概率提升 | 配合新角色上线 |
| **养成加速** | 某养成线消耗减半或产出翻倍 | 鼓励集中养成 |
| **掉落翻倍** | 特定副本掉落翻倍 | 引导刷指定副本 |
| **充值返利** | 充值额外赠送钻石/道具 | 配合版本更新 |
| **消费排行** | 活动期间消费前N名有额外奖励 | 刺激大R消费 |
| **累充活动** | 累计充值达档位获得奖励 | 档位设计要有吸引力 |
| **兑换活动** | 刷副本掉活动道具兑换奖励 | 鼓励在线时长 |
| **Boss rush** | 限时开放强力Boss，全服排名 | 促进付费和在线 |
| **答题/小游戏** | 知识问答、小游戏玩法 | 轻松休闲 |

### 2.2 活动日历
| 周期 | 活动类型 | 说明 |
|---|---|---|
| **每日** | 日常任务、每日副本 | 基础日常循环 |
| **每周** | 周常任务、周末双倍 | 周中积累，周末爆发 |
| **每月** | 月卡刷新、月度活动 | 月度节奏 |
| **节日** | 春节、国庆、情人节、圣诞 | 大型活动，奖励丰厚 |
| **版本** | 版本更新活动、新角色/玩法 | 最大规模活动 |
| **赛季** | PVP赛季、公会赛季 | 周期性重置 |

### 2.3 公会活动异常处理

设计公会相关活动（公会战、公会赛季、公会副本、公会排行榜）时，必须考虑成员变更导致的边界条件：

| 异常场景 | 问题描述 | 设计建议 |
|---|---|---|
| **活动期间退出公会** | 玩家在公会战/赛季进行中途退出，其已贡献的伤害/积分如何计算？ | 退出时锁定个人贡献，按退出前贡献结算奖励；或要求活动全程在公会内才能获得奖励 |
| **活动期间加入公会** | 新成员在活动进行中途加入，能否参与？奖励如何计算？ | 设置参与门槛（如加入满24小时/48小时）；或允许参与但奖励按实际贡献天数折算 |
| **公会战报名后人员变动** | 报名阶段和战斗阶段之间，公会成员数量/战力发生较大变化 | 报名时锁定参战名单和战力；战斗阶段不允许新成员参战 |
| **赛季结算前跳槽** | 玩家在高排名公会蹭完赛季奖励前退出，到低排名公会 | 赛季奖励按结算瞬间的公会成员身份发放；设置赛季末锁定机制（最后N天禁止退出/加入） |
| **公会解散/合并** | 赛季进行中，公会解散或合并，成员归属变化 | 解散前强制结算当前赛季奖励；合并时保留较高公会的赛季进度 |
| **恶意踢人** | 会长在奖励结算前恶意踢出高贡献成员，独占奖励 | 设置踢人冷却期（结算前N天禁止踢人）；或被踢成员仍按历史贡献获得应得奖励 |

## 3. 活动心流体验设计

> **定义**：基于心流理论和活动节奏设计方法论，评估活动方案的玩家体验曲线，识别心流失衡风险，优化活动节奏结构。
> **适用范围**：所有含限时活动的游戏

---

### 3.1 心流理论（Flow Theory）

#### 理论来源
Mihaly Csikszentmihalyi 《心流：最佳体验的心理学》

#### 心流八要素

| 要素 | 定义 | 在活动设计中的体现 |
|------|------|-------------------|
| 明确目标 | 玩家清晰知道要达成什么 | 签到天数目标、战令等级目标 |
| 即时反馈 | 行动后立即获得结果 | 任务完成提示、奖励到账动画 |
| 技能与挑战平衡 | 难度与能力匹配 | 副本分难度档次 |
| 行动与意识融合 | 沉浸于体验中 | 庆典氛围包装 |
| 全神贯注 | 注意力高度集中 | 多模块并行推进 |
| 掌控感 | 对局面有控制 | 战令解锁自主选择 |
| 自我意识消失 | 沉浸忘记时间 | 连续签到机制 |
| 时间扭曲 | 时间感改变 | 活动倒计时 |

#### 心流通道模型

心流产生于**技能**与**挑战**的平衡区间：
- 挑战过高 + 技能不足 → 焦虑
- 挑战过低 + 技能过高 → 无聊
- 挑战适中 + 技能匹配 → 心流

在活动设计中，通过动态调整活动难度/奖励，使不同层级的玩家都能进入心流状态。

---

### 3.2 活动节奏曲线设计

#### 节奏曲线类型

| 类型 | 特征 | 适用场景 |
|------|------|----------|
| **单峰型** | 只有一个主峰值 | 周年庆、版本更新 |
| **多峰型** | 多个递进的峰值 | 长周期活动（14天+） |
| **阶梯型** | 平稳上升→稳定→收尾 | 战令、赛季活动 |
| **爆发型** | 前期蓄力→瞬间释放 | 限时抽奖、Boss Rush |

#### 内容新鲜度模型

```
新鲜度指数 = 模块数量 / 活动天数

# 参考值
新鲜度 >= 1.0   # 节奏适中，内容充足
新鲜度 0.5-1.0  # 轻微重复感，需注意收尾
新鲜度 < 0.5    # 严重疲劳风险，必须增加模块或缩短周期
```

#### 多峰结构设计

| 节点类型 | 间隔建议 | 心流强度 | 设计要点 |
|----------|----------|----------|----------|
| 主峰值 | — | ⭐⭐⭐⭐⭐ | 不可替代的锚点（周年庆当天、版本更新日） |
| 次峰值1 | 主峰值后 4-7天 | ⭐⭐⭐⭐ | 递进式奖励/新内容解锁 |
| 次峰值2 | 次峰值1后 5-7天 | ⭐⭐⭐⭐ | 差异化玩法或社交事件 |
| 收尾峰值 | 活动结束前 3-5天 | ⭐⭐⭐ | 终点仪式、限时返场、最终冲刺 |

> **注意**：峰值之间必须有内容填充，避免"高峰后断崖"。

#### 里程碑设计方法论

| 里程碑类型 | 功能 | 设计要点 |
|------------|------|----------|
| **进度里程碑** | 量化目标可视化 | 战令等级、签到天数、累计积分 |
| **情感里程碑** | 唤起情怀共鸣 | 周年回顾、玩家故事、社区荣誉 |
| **竞争里程碑** | 激发排行动力 | 每日/每周排行榜、段位晋升 |
| **收尾里程碑** | 制造稀缺紧迫 | "最后X天"、倒计时、绝版奖励提示 |

---

### 3.3 活动心流诊断方法

#### 诊断流程

1. **模块心流价值评估**：对每个活动模块打分（1-5分），评估其对该活动心流通道的贡献
2. **时序分布绘制**：将模块按时间轴排列，绘制心流强度曲线
3. **失衡风险识别**：
   - 识别连续3天以上无新内容解锁的"空窗期"
   - 识别奖励密度骤降的"断崖期"
   - 识别参与度门槛过高的"阻塞点"
4. **多峰结构优化**：基于诊断结果，重新设计峰值节点和填充内容

#### 常见心流失衡模式

| 失衡模式 | 表现 | 解决方案 |
|----------|------|----------|
| **前高后低** | 开头几天内容过多，后期空虚 | 分散峰值，增加中后期内容 |
| **无峰平淡** | 全程节奏一致，缺乏高潮 | 设计明确的主峰值和次峰值 |
| **峰值过密** | 连续多天高强度，玩家疲劳 | 峰值间隔至少3-4天 |
| **阻塞断层** | 某模块门槛过高导致大量流失 | 降低门槛或提供替代路径 |
| **收尾仓促** | 最后几天无内容，玩家提前流失 | 增加收尾里程碑和返场机制 |

---

### 3.4 设计检查清单（心流专项）

- [ ] 活动是否有一个不可替代的主峰值？
- [ ] 长周期活动（14天+）是否有至少2个次峰值？
- [ ] 峰值间隔是否在3-7天之间？
- [ ] 内容新鲜度指数是否 >= 0.5？
- [ ] 是否存在连续3天以上的内容空窗期？
- [ ] 各层级玩家（免费/小R/中R/大R）是否都有心流入口？
- [ ] 里程碑设计是否覆盖进度、情感、竞争、收尾四类？
- [ ] 收尾阶段是否有足够的留存机制？

## 4. 赛季系统

### 4.1 赛季结构
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **赛季时长** | 通常4-8周 | 不宜过长或过短 |
| **赛季奖励** | 根据段位/排名发放 | 高段位奖励要足够诱人 |
| **赛季重置** | 段位部分重置 | 高段位重置后不会掉太多 |
| **赛季内容** | 新角色、新玩法、新皮肤 | 保持新鲜感 |

### 4.2 赛季设计要点
- **追赶机制**：新玩家/回归玩家有追赶途径
- **回归动力**：赛季重置提供重新竞争的机会
- **内容节奏**：赛季内容要足够支撑整个赛季

## 5. 版本更新系统

### 5.1 版本内容
| 内容 | 说明 | 设计要点 |
|---|---|---|
| **新角色** | 新可玩角色/英雄 | 配合卡池上线 |
| **新关卡** | 新主线/副本/活动关卡 | 扩展内容深度 |
| **新系统** | 新玩法/养成线 | 长期目标更新 |
| **平衡调整** | 角色/装备数值调整 | 保持游戏健康 |
| **优化修复** | 体验优化、bug修复 | 持续迭代 |

### 5.2 版本节奏
| 周期 | 内容规模 | 说明 |
|---|---|---|
| **小版本** | 2-4周 | 活动、小功能、优化 |
| **中版本** | 1-2月 | 新角色、新活动 |
| **大版本** | 3-6月 | 新系统、大型活动 |

### 5.3 版本预热
| 手段 | 说明 | 设计要点 |
|---|---|---|
| **前瞻直播** | 策划直播介绍新版本 | 建立预期 |
| **预热活动** | 版本前的小活动 | 维持热度 |
| **悬念营销** | 逐步放出版本信息 | 制造话题 |
| **预约奖励** | 版本预约奖励 | 拉动回流 |

## 6. 回归系统

### 6.1 回归判定
| 离线时间 | 回归类型 | 奖励梯度 |
|---|---|---|
| **7天** | 轻度回归 | 基础奖励 |
| **14天** | 中度回归 | 中等奖励 |
| **30天+** | 重度回归 | 丰厚奖励 |

### 6.2 回归内容
| 功能 | 说明 | 设计要点 |
|---|---|---|
| **回归任务** | 回归后7日内完成指定任务 | 引导重新熟悉游戏 |
| **回归buff** | 经验/掉落加成buff | 帮助快速追上 |
| **回归商店** | 回归专属商店，折扣出售 | 促进回归后付费 |
| **召回奖励** | 活跃玩家召回流失好友 | 双方获得奖励 |

## 7. 推送与通知系统

### 7.1 推送类型
| 类型 | 触发条件 | 设计要点 |
|---|---|---|
| **体力回满** | 体力达到上限时 | 提醒上线消耗 |
| **活动开始/结束** | 限时活动开始或即将结束 | 制造紧迫感 |
| **公会战提醒** | 公会战即将开始 | 促进社交参与 |
| **被攻击提醒** | PVP中被击败 | 促进复仇/调整阵容 |
| **福利提醒** | 免费抽卡/奖励可领取 | 低门槛吸引上线 |
| **回归召回** | 离线一段时间后 | 个性化文案 |

### 7.2 推送设计约束
- **频率控制**：过多会引起反感导致关闭推送
- **文案吸引**：要有吸引力，避免纯系统通知
- **用户设置**：玩家可选择接收哪些类型
