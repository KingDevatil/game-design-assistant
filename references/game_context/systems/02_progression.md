# 养成系统

> **定义**：给玩家提供长期目标，是留存和付费的核心驱动力。通过持续的成长反馈让玩家保持投入。
> **适用范围**：所有含角色/账号成长的游戏

---

## 1. 角色养成

### 1.1 等级系统
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **经验获取** | 任务、战斗、副本、挂机 | 多种途径，避免单一枯燥 |
| **升级曲线** | 前期快、中期稳、后期慢 | 总经验需求指数增长 |
| **等级上限** | 初始上限+版本更新提升 | 上限提升是版本重要内容 |
| **经验溢出** | 满级后经验转化为其他资源 | 避免经验浪费 |

### 1.2 突破/升星系统
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **突破材料** | 消耗特定材料提升等级上限 | 材料获取途径明确 |
| **升星机制** | 消耗同名角色/碎片提升星级 | 高星提升显著 |
| **稀有度** | N→R→SR→SSR→UR等 | 稀有度决定成长上限 |

### 1.3 觉醒/命座系统
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **觉醒效果** | 大幅提升属性和解锁新技能 | 质变点设计 |
| **命座机制** | 重复获得角色解锁命座 | 付费深度点 |
| **保底设计** | 硬保底/软保底机制 | 避免无限沉船 |

## 2. 装备/圣遗物系统

### 2.1 装备基础
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **装备部位** | 武器、头盔、铠甲、手套、鞋子、饰品 | 部位数量决定养成深度 |
| **品质分级** | 白/绿/蓝/紫/橙/红/神话 | 品质决定基础属性和词条数 |
| **基础属性** | 攻击力、防御力、生命值等 | 随装备等级和品质提升 |
| **随机词条** | 暴击、攻速、命中等副属性 | 随机范围决定毕业难度 |

### 2.2 装备强化
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **强化等级** | 消耗材料和金币提升等级 | 成功率设计 |
| **精炼/升星** | 消耗同名装备提升品质 | 高等级失败惩罚要谨慎 |
| **套装效果** | 穿戴多件同套装触发额外属性 | 2件/4件/6件套效果递进 |
| **镶嵌/附魔** | 宝石镶嵌、附魔附加特效 | 提供额外自定义空间 |

### 2.3 圣遗物/遗器系统（随机词条型）
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **主词条** | 固定类型，随等级提升数值 | 部位决定主词条范围 |
| **副词条** | 随机生成，强化时随机提升 | 4-5条副词条 |
| **套装效果** | 特定组合触发强力效果 | 驱动玩家刷取 |
| **毕业难度** | 完美词条概率极低 | 控制毕业周期 |

**典型案例**：《原神》圣遗物、《崩坏：星穹铁道》遗器、《明日方舟》模组

## 3. 技能系统

### 3.1 技能类型
| 类型 | 说明 | 设计要点 |
|---|---|---|
| **主动技能** | 玩家主动释放的技能 | 每个角色3-6个，有冷却和消耗 |
| **被动技能** | 常驻生效的属性加成或特效 | 提供Build多样性 |
| **大招/奥义** | 高伤害或强力效果的终极技能 | 需要积攒能量/怒气释放 |
| **连携技能** | 特定角色组合触发的特殊技能 | 鼓励收集特定组合 |

### 3.2 技能升级
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **升级消耗** | 消耗技能书/金币提升等级 | 每级提升伤害倍率或减冷却 |
| **技能树** | 分支选择，不同路线不同风格 | 重置成本要合理 |
| **连招系统** | 技能间可衔接形成Combo | 操作上限和爽感的来源 |

## 4. 天赋/潜能系统

### 4.1 天赋树设计
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **分支选择** | 多分支天赋，点数有限需取舍 | 设计要有明显的流派差异 |
| **天赋点获取** | 升级、任务、成就、特殊道具 | 获取速度控制Build成型节奏 |
| **重置机制** | 允许重置天赋重新加点 | 免费或低成本重置鼓励尝试 |
| **通用vs专属** | 全角色通用天赋+角色专属天赋 | 通用提供基础，专属提供特色 |

---

## 5. 羁绊/好感系统

### 5.1 好感度系统
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **提升方式** | 送礼、互动、共同出战 | 多种途径提升 |
| **等级解锁** | 好感等级解锁剧情/语音/属性 | 奖励要有吸引力 |
| **专属剧情** | 高好感度解锁角色专属故事 | 增强情感投入 |
| **属性加成** | 好感度转化为全队属性 | 轻度数值驱动 |

### 5.2 羁绊系统
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **组合加成** | 特定角色同时在场触发加成 | 鼓励收集特定系列 |
| **羁绊等级** | 共同出战提升羁绊等级 | 长期养成目标 |
| **剧情解锁** | 羁绊等级解锁组合剧情 | 增强世界观 |

---

## 6. 收集/图鉴系统

### 6.1 图鉴类型
| 类型 | 说明 | 设计要点 |
|---|---|---|
| **角色图鉴** | 收集角色解锁图鉴 | 收集进度转化为全队属性 |
| **装备图鉴** | 收集装备解锁 | 曾拥有即可解锁 |
| **怪物图鉴** | 击败怪物解锁信息和掉落 | 增加探索动力 |
| **风景/剧情图鉴** | 到达地点/完成剧情解锁 | 增强世界观沉浸感 |

### 6.2 成就系统
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **成就分类** | 战斗、收集、社交、探索、挑战 | 覆盖游戏各方面 |
| **成就等级** | 铜/银/金/白金，完成度递进 | 如"击败100/1000/10000只怪物" |
| **成就奖励** | 属性加成、称号、头像框、货币 | 奖励要有实际价值 |
| **隐藏成就** | 不公开条件，完成后解锁 | 增加探索乐趣 |

### 6.3 称号系统
| 要素 | 说明 | 设计要点 |
|---|---|---|
| **展示性称号** | 纯展示，无属性 | 稀有称号是身份象征 |
| **属性称号** | 带属性加成 | 提供轻度数值驱动 |
| **获取途径** | 成就、活动、排名、特殊条件 | 多样化获取 |

---

## 7. 设计检查清单

- [ ] 养成深度是否足够？是否有明确的长期目标？
- [ ] 养成曲线是否平滑？是否存在明显的卡点？
- [ ] 免费玩家是否能通过时间弥补付费差距？
- [ ] 随机性是否适度？挫败感与惊喜感的平衡？
- [ ] 毕业周期是否合理？是否会导致玩家流失？
- [ ] 是否有足够的收集动力？图鉴奖励是否有吸引力？
