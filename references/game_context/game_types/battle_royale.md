# 游戏类型：Battle Royale（大逃杀）

> **对标参考**：《PUBG》、《APEX英雄》、《堡垒之夜》、《使命召唤：战区》、《Free Fire》  
> **核心标签**：生存竞技、缩圈机制、搜刮装备、百人同场、最后存活

---

## 核心循环

```
跳伞降落 → 搜刮武器/装备/物资 → 转移/交战 → 缩圈推进 → 决赛圈激战 → 吃鸡/出局
```

## 系统架构特征

| 系统模块 | 设计要点 |
|---|---|
| **跳伞机制** | 飞机/巴士航线随机，玩家自主选择降落点，热点区域资源丰富但危险 |
| **搜刮系统** | 武器、护甲、医疗包、投掷物、配件，随机分布在地图各处 |
| **缩圈机制** | 安全区逐渐缩小，毒圈/风暴伤害递增，迫使玩家向中心移动 |
| **复活机制** | 部分游戏有复活队友机制（APEX重生信标、战区古拉格） |
| **载具系统** | 车辆/摩托/船等快速移动工具，增加战术选择 |
| **单排/组队** | 单排、双排、四排，组队模式强调配合和分工 |
| **赛季系统** | 通行证、段位、赛季主题地图/机制 |
| **付费设计** | 皮肤（角色/武器/降落伞/载具）、通行证、表情动作，**无属性付费** |

## 核心设计要素

| 要素 | 说明 | 设计要点 |
|---|---|---|
| **地图设计** | 大地图（4x4km~8x8km），多样地形（城市/荒野/山地/水域） | 资源分布、热点区域、转移路线 |
| **缩圈节奏** | 圈速、圈伤害、圈间隔决定游戏节奏 | 太快=运气成分大，太慢=节奏拖沓 |
| **枪械平衡** | 不同距离和场景有最优武器选择 | 霰弹近战强、狙击远程强、步枪全能 |
| **信息博弈** | 枪声、脚步声、载具声暴露位置 | 声音设计是核心竞争力 |
| **物资分级** | 白/绿/蓝/紫/橙品质，高级物资在高级区域或空投 | 风险与收益匹配 |

## 关键设计约束

1. **对局时长**：控制在15-30分钟，太长不适合移动端
2. **匹配公平**：新手不应匹配到高手，需设计隐藏分和段位保护
3. **外挂问题**：透视、自瞄、加速严重破坏体验，反作弊必须强力
4. **服务器性能**：百人同场对服务器压力大，需优化网络同步
5. **随机性管理**：搜刮随机性带来乐趣，但不能让运气完全决定胜负
6. **社交体验**：野排队友体验差（不配合、抢物资），需优化组队匹配

## 常见策划产出物
- 地图设计文档（资源分布、热点、缩圈中心概率）
- 武器设计表（伤害、射速、后坐力、弹匣、稀有度）
- 缩圈参数设计（圈速、伤害、间隔、随机中心算法）
- 载具设计（速度、耐久、座位、刷新点）
- 反作弊方案（检测逻辑、举报系统、惩罚机制）
