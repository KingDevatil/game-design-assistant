# 游戏类型：Card Game（卡牌游戏）

> **对标参考**：《炉石传说》、《万智牌》、《游戏王》、《影之诗》、《Slay the Spire》（卡牌Roguelike）、《Marvel Snap》  
> **核心标签**：卡牌构筑（Deck Building）、策略对抗、资源管理、回合制、收集养成

---

## 核心循环

```
收集卡牌 → 构筑牌组 → 对战 → 调整牌组 → 获取新卡/资源 → 优化构筑
```

## 主要子类型

| 子类型 | 特征 | 代表 |
|---|---|---|
| **CCG（集换式卡牌）** | 收集实体/虚拟卡牌，构筑牌组对战，可交易 | 《万智牌》、《炉石传说》 |
| **TCG（ trading card game）** | 强调卡牌交易和经济系统 | 同上 |
| **LCG（成长式卡牌）** | 固定扩展包，无随机抽卡 | 《Arkham Horror LCG》 |
| **Deckbuilding Roguelike** | 单局内从基础牌组逐步构建 | 《杀戮尖塔》、《怪物火车》 |
| **Auto Battler（自走棋）** | 卡牌/棋子自动战斗，玩家负责搭配和站位 | 《云顶之弈》、《金铲铲之战》 |

## 系统架构特征

| 系统模块 | 设计要点 |
|---|---|
| **卡牌设计** | 费用、攻击力、生命值、特效、关键词（战吼、亡语、连击等） |
| **资源系统** | 法力水晶/费用，每回合增长，限制出牌节奏 |
| **牌组构筑** | 最小/最大牌数限制，同名卡数量限制，围绕核心Combo构筑 |
| **对战系统** | 回合制，抽牌→出牌→攻击→结束，先降对方生命为0者胜 |
| **收集系统** | 开包/抽卡获取新卡，稀有度分级（普通/稀有/史诗/传说） |
| **赛季/扩展** | 定期推出新扩展包，引入新机制，旧卡退环境（轮换） |
| **付费设计** | 卡包购买、通行证、皮肤/卡背/特效、预组卡组 |

## 卡牌设计核心原则

| 原则 | 说明 | 示例 |
|---|---|---|
| **费用曲线** | 低费卡灵活但弱，高费卡强力但慢 | 1费1/1，8费8/8+特效 |
| **卡牌效用** | 单卡强度要合理，避免"必带卡" | 设计替代方案和Counter卡 |
| **Combo设计** | 卡牌间联动产生强力效果 | A卡+B卡=额外效果 |
| **Counter机制** | 有矛必有盾，保持生态平衡 | 快攻→控制→Combo→快攻 |
| **随机性控制** | 抽牌随机带来变数，但不能完全看脸 | 过牌手段、检索机制 |

## 关键设计约束

1. **Pay-to-Win控制**：集卡类游戏容易变成"谁花钱多谁赢"，需设计免费获取途径和平衡机制
2. **Meta固化**：环境容易被某几套牌组主导，需频繁平衡调整
3. **学习门槛**：大量卡牌效果和关键词对新玩家不友好，需设计简化模式
4. **对局时长**：控制在5-20分钟，太长影响移动端体验
5. **退环境争议**：旧卡退环境可保持新鲜感，但会引起玩家不满

## 常见策划产出物
- 卡牌设计文档（费用、效果、稀有度、插画需求）
- 扩展包设计（新机制、主题、卡牌列表）
- 平衡性调整方案（胜率、使用率分析）
- 新手引导设计（教学关卡、预组卡组）
- 经济系统设计（卡包定价、粉尘/合成系统）
