# 游戏类型：横版街机格斗 + 多角色 + 数值养成

> **对标参考**：《火影忍者》手游、《拳皇命运》、《街霸：对决》、《三国战纪》系列  
> **核心标签**：横版卷轴、即时格斗、角色收集、技能连招、数值养成

---

## 核心循环

```
获取角色 → 养成强化（升级/升星/装备/技能） → 挑战副本/PVP → 获取资源 → 养成更强角色
```

## 系统架构特征

| 系统模块 | 设计要点 |
|---|---|
| **战斗系统** | 横版卷轴即时战斗，虚拟摇杆+技能按键，支持普攻连段+技能取消+奥义终结 |
| **角色系统** | 多角色收集（抽卡/碎片合成/活动获取），每个角色独立养成线 |
| **技能系统** | 普攻+主动技能（2-4个）+奥义/必杀技，技能可升级解锁新效果 |
| **数值养成** | 等级、星级、装备、技能等级、天赋/潜能、专属武器/羁绊 |
| **PVE内容** | 主线剧情关卡、精英副本、材料副本、限时挑战、Boss Rush |
| **PVP内容** | 实时1v1竞技场、段位赛、3v3团队战、公会战 |
| **经济系统** | 多货币（金币、钻石、体力、竞技币、公会币），体力限制日常进度 |
| **社交系统** | 公会、好友助战、排行榜、师徒、组队副本 |
| **付费设计** | 角色抽卡、月卡、战令/通行证、限时礼包、皮肤外观 |

## 战斗系统详解

### 操作框架
- **移动**：虚拟摇杆左右移动，支持跑动、冲刺、后撤
- **普攻**：连打触发多段普攻，最后一段通常带击飞/倒地
- **技能**：2-4个主动技能，有冷却时间和能量/怒气消耗
- **奥义/必杀**：高伤害大招，需积攒怒气/能量释放，通常带特写动画
- **防御/闪避**：格挡、闪避、受身等防御操作

### 核心战斗机制
| 机制 | 说明 | 设计要点 |
|---|---|---|
| **连招系统** | 普攻→技能→奥义的连段组合 | 每角色设计2-3套基础连招路线 |
| **技能取消** | 用技能取消普攻后摇，实现无缝连段 | 取消窗口帧数决定操作上限 |
| **霸体/无敌帧** | 部分技能释放期间免疫控制/伤害 | 平衡进攻与防守节奏 |
| **倒地保护** | 连续受击后触发保护机制，防止无限连 | 保护触发条件和持续时间 |
| **属性克制** | 角色/技能属性间的克制关系 | 火克风、风克雷等（根据题材调整） |
| **援护/助战** | 召唤后备角色释放技能辅助战斗 | 援护冷却和技能选择 |

### 手感调优关键参数
- 普攻前后摇帧数
- 技能前摇/判定帧/后摇
- 受击硬直时间
- 击飞高度和距离
- 落地反弹机制

## 角色系统设计

### 角色定位分类
| 定位 | 特征 | 代表风格 |
|---|---|---|
| **突进型** | 高机动、快速近身、连招能力强 | 刺客风格 |
| **力量型** | 高伤害、慢攻速、范围大 | 狂战士风格 |
| **远程型** | 远距离输出、控制能力强 | 法师/射手风格 |
| **防御型** | 高生存、反打能力强、控制多 | 坦克风格 |
| **均衡型** | 无明显短板，适应性强 | 战士风格 |

### 角色养成维度（单武将）
```
角色战力 = f(等级, 星级, 装备评分, 技能等级, 天赋等级, 羁绊加成, 专属武器)
```

| 养成线 | 说明 | 资源来源 |
|---|---|---|
| 等级 | 基础属性成长 | 经验副本、日常任务 |
| 星级 | 解锁新技能/提升属性系数 | 角色碎片（抽卡/副本/活动） |
| 装备 | 6件套，可强化/精炼/附魔 | 副本掉落、商店、锻造 |
| 技能 | 主动技能+被动技能升级 | 技能书、金币 |
| 天赋/潜能 | 属性分支选择 | 专属材料 |
| 羁绊 | 特定角色组合触发加成 | 收集对应角色 |
| 专属武器 | 角色专属，大幅强化特性 | 活动/高难副本 |

### 常规数值养成系统（全局/账号级）
> **说明**：除了单个武将的独立养成线外，项目中可能还存在**全局/账号级别的数值养成系统**，其属性对所有上阵角色均生效。
>
> **重要**：本知识库不预设任何具体的全局养成系统。当用户使用本Skill涉及全局养成相关内容时，应通过引导式对话向用户确认以下信息：
> 1. 当前项目是否有全局养成系统？如有，请列出系统名称。
> 2. 每个全局养成系统提供的属性加成类型是什么？
> 3. 全局养成系统的培养方式和资源来源是什么？
> 4. 全局养成对战斗的影响程度（轻度/中度/重度）？
>
> 在用户确认之前，所有涉及全局养成的设计输出中应标注"【待确认：全局养成系统】"，不做任何假设性设计。

### 养成系统对战斗的影响
- **单武将养成**：决定个体强度差异，是PVE推图和PVP竞技的核心变量
- **全局养成**（如有，需用户确认）：提供基础战力保障，确保即使更换上阵角色也有稳定战斗力
- **设计原则**：全局养成（如有）提供"保底战力"，单武将养成提供"突破上限"，两者相辅相成
- **使用约束**：涉及全局养成时，必须先通过对话确认用户项目中的实际系统，禁止无中生有

## 关键设计约束

1. **操作门槛**：移动端虚拟按键的操作精度有限，连招判定要足够宽松
2. **数值平衡**：养成深度大的同时，PVP必须做数值平衡（如统一属性模板或属性压缩）
3. **内容消耗**：角色养成周期长，需持续推出新角色和新玩法维持新鲜感
4. **付费平衡**：抽卡是核心付费点，需保证保底机制和免费获取途径
5. **网络同步**：实时PVP对网络延迟敏感，需优化同步方案和断线重连

## 常见策划产出物
- 角色设计文档（定位、技能、数值框架、背景）
- 战斗系统PRD（操作逻辑、帧数据、平衡规则）
- 关卡设计文档（敌人配置、节奏、难度曲线）
- 数值配置表（角色成长、装备属性、经济产出）
- 活动方案（限时角色、养成加速、PVP赛季）
