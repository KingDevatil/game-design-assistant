# 游戏类型：ARPG（动作角色扮演游戏）

> **对标参考**：《暗黑破坏神》系列、《流放之路》、《火炬之光》、《崩坏3》、《战双帕弥什》  
> **核心标签**：即时动作战斗、装备驱动（Loot-driven）、Build构筑、割草爽感、重复可玩性

---

## 核心循环

```
选择职业/角色 → 刷图获取装备 → 优化Build（技能+装备词缀） → 挑战更高难度 → 获取更强Loot
```

## 系统架构特征

| 系统模块 | 设计要点 |
|---|---|
| **战斗系统** | 即时动作战斗，强调走位、闪避、技能Combo，割草式清怪体验 |
| **职业/角色系统** | 多职业选择，每个职业有独特的技能树和战斗风格 |
| **装备系统** | 装备驱动核心，白/绿/蓝/紫/橙/暗金品质分级，随机词缀（Affix） |
| **Build构筑** | 技能搭配 + 装备词缀 + 符文/宝石 + 天赋树，形成无限组合 |
| **难度系统** | 多难度层级（普通/困难/地狱/大秘境），难度越高掉落越好 |
| **赛季/轮回系统** | 定期重置赛季，新机制/新装备/排行榜，保持新鲜感 |
| **经济系统** | 单一或双货币，装备交易/拍卖行（视设计而定），材料合成 |
| **付费设计** | 角色/职业解锁、外观皮肤、便利性道具（背包/仓库扩展）、赛季通行证 |

## 战斗系统详解

### 核心战斗机制
| 机制 | 说明 | 设计要点 |
|---|---|---|
| **割草爽感** | 面对大量杂兵时的快速清场体验 | 技能范围、伤害数字反馈、连击数 |
| **Boss战** | 精英/Boss具备独特机制和阶段 | 攻击前摇提示、阶段转换、弱点暴露 |
| **闪避/防御** | 无敌帧闪避或格挡系统 | 闪避资源管理（体力/次数） |
| **技能Build** | 主动技能+被动技能+装备特效联动 | 设计足够多的联动可能性 |
| **元素/异常状态** | 火/冰/电/毒等元素，附加燃烧/冰冻/感电/中毒 | 元素反应/叠加机制 |

### 装备词缀设计（Loot核心）
| 词缀类型 | 示例 | 设计原则 |
|---|---|---|
| **基础属性** | 攻击力、防御力、生命值 | 随装备品质和等级提升 |
| **核心词缀** | 暴击率、攻击速度、技能伤害加成 | 决定Build方向 |
| **特殊特效** | 击杀回复生命、技能触发额外效果 | 改变玩法体验 |
| **套装效果** | 穿戴多件同名装备触发强力特效 | 鼓励收集完整套装 |

## 关键设计约束

1. **Loot节奏**：掉落频率和品质要形成正向反馈循环，避免"肝而无获"
2. **Build平衡**：避免单一Build过于强势，保持多样性，但也要允许"版本答案"
3. **操作门槛**：动作战斗的操作复杂度要与目标用户匹配，移动端简化操作
4. **内容消耗**：后期内容依赖赛季更新和终局玩法（Endgame）维持
5. **数值膨胀**：多赛季累积容易导致数值失控，需设计赛季独立的数值体系

## 常见策划产出物
- 职业设计文档（技能树、定位、数值框架）
- 装备词缀设计表（词缀池、权重、品质分布）
- 关卡/副本设计（怪物配置、密度、节奏）
- 赛季设计文档（新机制、新装备、赛季主题）
- 数值配置表（伤害公式、成长曲线、掉落概率）
