# 游戏类型：Idle / AFK（放置/挂机游戏）

> **对标参考**：《剑与远征》、《放置少女》、《Idle Heroes》、《点击英雄》、《最强蜗牛》  
> **核心标签**：离线收益、轻度操作、数值成长、长周期养成、碎片化时间

---

## 核心循环

```
放置挂机获取资源 → 升级角色/装备 → 推图/爬塔 → 挂机效率提升 → 循环
```

## 系统架构特征

| 系统模块 | 设计要点 |
|---|---|
| **挂机收益** | 离线也能获得资源（金币、经验、材料），在线收益略高于离线 |
| **推图/爬塔** | 主线关卡+无限爬塔，战力达标自动推进，卡关需养成 |
| **角色养成** | 大量角色收集，升级、升星、装备、技能，深度数值养成 |
| **阵容搭配** | 上阵多个角色，考虑职业/种族/羁绊加成，策略性有限但存在 |
| **日常任务** | 少量日常（5-15分钟），奖励丰厚，鼓励每日登录 |
| **赛季/活动** | 限时活动、新角色UP、赛季排行榜 |
| **付费设计** | 月卡、战令、角色抽卡、加速道具、资源礼包，**付费可显著加速进度** |

## 核心设计要素

| 要素 | 说明 | 设计要点 |
|---|---|---|
| **离线收益** | 玩家下线后角色继续战斗/采集 | 离线时长上限、收益衰减 |
| **自动战斗** | 战斗全自动，玩家只需战前配置 | 可观看战斗或跳过 |
| **战力检测** | 战力数值直观反映强度，卡关时明确提示差距 | 战力=属性总和+羁绊加成 |
| **抽卡收集** | 角色主要通过抽卡获取，稀有度驱动收集欲 | 保底、UP池、限定角色 |
| **社交竞争** | 排行榜、公会、轻度PVP | 竞争压力要适度，不强迫 |

## 关键设计约束

1. **付费深度**：放置类游戏付费上限可以很高（氪佬和平民差距大），但要保证免费玩家能玩
2. **内容消耗**：后期容易进入"数值膨胀+等新内容"状态，需设计足够多的养成线和系统
3. **广告变现**：部分放置游戏依赖广告变现，广告频率和奖励要平衡
4. **留存设计**：签到、七日登录、回归奖励、推送通知，多种手段维持留存
5. **新手节奏**：前期进度快、奖励多，让玩家快速体验到成长爽感

## 常见策划产出物
- 角色设计文档（属性、技能、定位、稀有度）
- 数值成长曲线（升级消耗、属性成长、战力公式）
- 关卡难度设计（战力门槛、敌人配置、奖励）
- 抽卡概率设计（概率表、保底机制、期望值）
- 付费点设计（礼包性价比、成长加速效果）
