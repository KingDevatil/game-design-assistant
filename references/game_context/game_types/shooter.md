# 游戏类型：Shooter（射击游戏）

> **对标参考**：《CS2》、《Valorant》、《APEX英雄》、《PUBG》、《使命召唤》、《守望先锋》  
> **核心标签**：枪法、反应速度、战术配合、地图控制、竞技对抗

---

## 核心循环

```
选择武器/角色 → 对枪/战术推进 → 击杀/占点/完成任务 → 回合/对局结算 → 获取奖励 → 解锁新内容
```

## 主要子类型

| 子类型 | 特征 | 代表 |
|---|---|---|
| **FPS（第一人称）** | 第一人称视角，强调精准瞄准 | 《CS2》、《Valorant》 |
| **TPS（第三人称）** | 第三人称视角，可利用掩体和视角优势 | 《PUBG》、《Gears of War》 |
| **Battle Royale（大逃杀）** | 大规模地图，生存到最后，缩圈机制 | 《PUBG》、《APEX》、《堡垒之夜》 |
| **Hero Shooter** | 角色有不同技能，结合射击和MOBA元素 | 《守望先锋》、《Valorant》 |
| **Tactical Shooter** | 强调战术配合、一枪毙命、经济系统 | 《CS2》、《彩虹六号》 |
| **Loot Shooter** | 射击+装备掉落+Build构筑 | 《命运2》、《全境封锁》 |

## 系统架构特征

| 系统模块 | 设计要点 |
|---|---|
| **武器系统** | 多类型武器（步枪/冲锋枪/狙击/霰弹/手枪/近战），每种有独特的后坐力、射速、伤害 |
| **角色/技能** | Hero Shooter中每个角色有独特技能（位移/控制/治疗/侦察） |
| **地图设计** | 对称/非对称地图，控制点位，高低差，掩体分布，视野线 |
| **经济系统** | 回合制购买武器/道具（CS/Valorant），或局内拾取（BR） |
| **匹配系统** | 基于MMR的匹配，段位系统，反作弊 |
| **付费设计** | 皮肤（武器/角色）、通行证、贴纸/挂饰，**无属性付费** |

## 射击游戏核心机制

| 机制 | 说明 | 设计要点 |
|---|---|---|
| **后坐力（Recoil）** | 连续射击时准星偏移 | 每种武器独特的后坐力模式 |
| **弹道/命中判定** | 即时命中（Hitscan）vs 物理弹道（Projectile） | 步枪Hitscan，火箭筒Projectile |
| **爆头机制** | 命中头部造成额外伤害 | 鼓励精准瞄准 |
| **移动与射击** | 移动中射击精度下降，急停恢复精度 | 增加身法技巧深度 |
| **经济回合** | 胜方奖励多，败方有连败补偿 | 让劣势方有机会翻盘 |
| **缩圈机制（BR）** | 安全区逐渐缩小，迫使玩家相遇 | 圈速、伤害、道具设计 |

## 关键设计约束

1. **反作弊**：射击游戏是外挂重灾区，反作弊是生命线
2. **网络同步**：延迟补偿（Lag Compensation）、Tick Rate直接影响手感
3. **TTK（Time To Kill）**：击杀时间决定游戏节奏，短TTK偏战术，长TTK偏操作
4. **平衡性**：武器/角色不能过强，地图不能偏向某一方
5. **竞技与娱乐平衡**：既要满足硬核竞技玩家，也要有休闲模式

## 常见策划产出物
- 武器设计文档（伤害、射速、后坐力、弹匣、价格）
- 角色设计文档（技能、定位、数值、背景）
- 地图设计文档（点位、通道、视野、出生点）
- 经济系统设计（奖励、价格、连败补偿）
- 反作弊方案（检测逻辑、举报系统、惩罚机制）
