# 游戏类型：Strategy / SLG（策略游戏 / 模拟游戏）

> **对标参考**：《三国志·战略版》、《率土之滨》、《文明》系列、《全面战争》系列、《XCOM》、《火焰纹章》  
> **核心标签**：资源管理、战术/战略决策、兵种克制、地图控制、长线养成

---

## 核心循环

```
资源采集 → 城市建设/兵种训练 → 科技研究 → 扩张/战争 → 联盟外交 → 争夺霸业
```

## 主要子类型

| 子类型 | 特征 | 代表 |
|---|---|---|
| **4X策略** | 探索（eXplore）、扩张（eXpand）、开发（eXploit）、消灭（eXterminate） | 《文明》、《群星》 |
| **RTS（即时战略）** | 实时操作，资源+建造+战斗同时进行 | 《星际争霸》、《帝国时代》 |
| **TBS（回合制策略）** | 回合制操作，强调深思熟虑 | 《文明》、《XCOM》 |
| **SLG（手游策略）** | 大地图沙盘、联盟战争、赛季制、强社交 | 《三国志·战略版》、《率土之滨》 |
| **战棋（SRPG）** | 格子地图、兵种克制、角色养成 | 《火焰纹章》、《梦幻模拟战》 |
| **塔防（TD）** | 建造防御塔阻止敌人进攻 | 《王国保卫战》、《明日方舟》 |

## 系统架构特征（以手游SLG为例）

| 系统模块 | 设计要点 |
|---|---|
| **大地图沙盘** | 无缝大地图，地块占领，资源点争夺，真实地形影响 |
| **城市建设** | 主城建筑升级，科技树，资源建筑（农田/伐木/矿场） |
| **武将/英雄系统** | 历史/ fantasy 角色收集，养成（等级/星级/技能），带兵打仗 |
| **兵种系统** | 步兵/骑兵/弓兵/器械等，兵种克制（骑克弓、弓克步、步克骑） |
| **联盟系统** | 玩家组成联盟，共享视野，协同作战，攻城略地 |
| **赛季系统** | 定期赛季重置，新地图、新规则、霸业结算 |
| **付费设计** | 武将抽卡、加速道具、资源包、月卡、赛季通行证 |

## 战棋/塔防核心机制

| 机制 | 说明 | 设计要点 |
|---|---|---|
| **格子移动** | 角色在格子地图上移动，移动力有限 | 地形影响移动力和属性 |
| **兵种克制** | 不同兵种间有克制关系 | 克制关系要清晰，但不能绝对 |
| **范围攻击** | 技能/攻击有范围效果 | AOE范围、友军伤害（可选） |
| **地形利用** | 高地、森林、河流等地形影响战斗 | 增加策略深度 |
| **塔防路径** | 敌人沿固定路径移动，玩家建造防御塔 | 路径设计、塔位设计 |

## 关键设计约束

1. **时间消耗**：SLG是重时间投入的游戏，需设计离线收益和自动化功能
2. **付费与公平**：付费玩家加速发展，但不能让免费玩家完全无法抗衡
3. **社交压力**：联盟战争可能带来强制上线压力，需设计合理的参与门槛
4. **赛季重置**：重置太频繁伤玩家，太久导致固化，需找到平衡点
5. **新手保护**：新玩家容易被老玩家碾压，需设计保护区/匹配机制

## 常见策划产出物
- 武将/英雄设计文档（属性、技能、兵种适性）
- 兵种设计表（攻防属性、克制关系、cost）
- 地图设计（资源分布、地形、城池位置）
- 赛季规则设计（霸业条件、结算奖励、新机制）
- 科技树设计（前置条件、效果、研究时间）
