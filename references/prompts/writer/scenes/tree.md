# Scene: WRI-tree - 对话树设计

## 场景标识
- **ID**: WRI-tree
- **名称**: 对话树设计
- **岗位**: 文案策划
- **优先级**: P0（Phase 2）

## 场景目的
输入剧情节点和分支意图，设计完整的对话树结构，包括：对话选项、条件判断、后果分支、回环机制，确保剧情逻辑自洽且有多周目价值。

## 输入模板

```markdown
## 对话树设计输入

### 剧情节点
- 节点名称：_______
- 剧情背景：_______
- 当前局势：_______

### 分支意图
| 选项方向 | 玩家表达 | 预期后果 |
|---|---|---|
| A | 忠义/强硬 | _______ |
| B | 权谋/圆滑 | _______ |
| C | 中立/观望 | _______ |

### 约束条件
- 最大深度：_______层
- 是否允许回环：是/否
- 是否有隐藏选项：是/否
- 选项是否有属性/好感度要求：_______
```

## 处理逻辑

1. **根节点设计**
   - 设计开场白，交代当前情境
   - 确保玩家理解为什么需要做出选择

2. **选项设计**
   - 每个选项有明确的价值取向差异
   - 选项文案有吸引力，不剧透后果
   - 隐藏选项的设计（需满足特定条件才显示）

3. **后果分支**
   - 每个选择导致不同的NPC反应
   - 短期后果（即时反馈）和长期后果（影响后续剧情）

4. **回环与收敛**
   - 设计必要的回环（不同选择最终到达同一节点）
   - 避免过度发散导致内容爆炸

5. **逻辑校验**
   - 检查是否存在死胡同（选择后无后续）
   - 检查条件判断是否合理
