# Scene: LVL-variant - 关卡变体批量生成

## 场景标识
- **ID**: LVL-variant
- **名称**: 关卡变体批量生成
- **岗位**: 关卡策划
- **优先级**: P1（Phase 2）

## 场景目的
输入一个基础关卡设计，通过调整参数（敌人、机关、时间、限制条件）批量生成多个难度/玩法变体，用于日常挑战、活动副本、无尽模式等场景。

## 输入模板

```markdown
## 关卡变体生成输入

### 基础关卡
- 关卡名称：_______
- 原关卡配置：_______

### 变体类型
- [ ] 难度变体（简单/困难/噩梦）
- [ ] 玩法变体（限时/限血/禁技能）
- [ ] 环境变体（天气/昼夜/地形）
- [ ] 敌人变体（全换某一类型/增加精英）

### 生成数量
- 需要生成的变体数：_______

### 约束条件
- 变体间差异度要求：_______（如：至少30%参数不同）
- 禁止出现的组合：_______
```

## 处理逻辑

1. **参数提取**
   - 从基础关卡提取可调整参数列表
   - 标注每个参数的调整范围和步长

2. **变体规则设计**
   - 为每种变体类型设计调整规则
   - 确保变体有明确的差异化体验

3. **批量生成**
   - 按规则生成指定数量的变体
   - 确保变体间不重复、有区分度

4. **质量校验**
   - 检查变体是否合理（不会出现不可能通关的情况）
   - 检查变体差异度是否达标
