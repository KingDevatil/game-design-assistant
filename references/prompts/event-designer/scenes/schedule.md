# Scene: EVT-schedule - 多活动并发排期

## 场景标识
- **ID**: EVT-schedule
- **名称**: 多活动并发排期
- **岗位**: 活动策划
- **优先级**: P1（Phase 2）
- **适用场景**: 当你需要同时规划和排期多个活动，避免冲突和玩家疲劳时。

## 场景目的

输入多个活动的计划信息和约束条件，检测时间/用户/资源冲突，评估玩家疲劳累积，输出优化后的排期方案和冲突解决建议。

## 输入模板

```markdown

### 约束条件
- 必须上线的活动：_______
- 不可重叠的活动：_______

- 付费玩家同时可参与活动上限：_______
```

## 处理逻辑

1. **时间冲突检测**
   - 绘制所有活动的时间轴分布
   - 识别直接时间重叠的活动
   - 标注必须错峰的关键活动

2. **用户群体冲突**
   - 分析同一时间段内面向相同用户群体的活动数量
   - 识别"活动轰炸"风险（同一用户被多个活动同时拉取注意力）
   - 评估不同层级玩家（免费/小R/中R/大R）的参与负担

3. **资源冲突检测**
   - 统计每个时间段的开发/美术/运营人力需求
   - 识别人力超负载的"资源瓶颈期"
   - 标注因资源限制必须延期的活动

4. **玩家疲劳评估**
   - 计算每日/每周的活动密度（活动数量 / 天数）
   - 识别连续高强度活动周（每周 >= 3个活动为高风险）
   - 评估活动空窗期（连续3天无任何活动为低风险但需关注）

5. **排期优化建议**
   - **错峰**：将面向相同用户群体的活动错开1-2周
   - **合并**：将小型活动合并为大型综合活动，减少运营负担
   - **拆分**：将过长的大型活动拆分为两段，制造两次峰值
   - **删减**：建议砍掉优先级低且资源冲突严重的活动
   - **填充**：在空窗期安排轻量级日常活动维持热度

6. **输出排期表**
   - 优化后的活动甘特图（ASCII）
   - 每周活动密度分布
   - 关键里程碑和冲突点标注
