# Scene: EVT-duration - 活动周期调整影响评估

## 场景标识
- **ID**: EVT-duration
- **名称**: 活动周期调整影响评估
- **岗位**: 活动策划
- **优先级**: P0（Phase 2）
- **适用场景**: 当你需要评估活动周期延长或缩短（如20天→30天）对玩家体验、付费、留存的影响时。

## 场景目的

输入现有活动方案和周期调整方案，量化评估周期变化对内容新鲜度、玩家疲劳曲线、付费节奏、留存拐点的综合影响，输出调整建议和风险预警。

## 处理逻辑

1. **周期对比分析**
   - 对比原周期和新周期的阶段结构变化
   - 计算各阶段天数变化比例
   - 识别阶段结构失衡风险

2. **内容新鲜度重算**
   - 原新鲜度指数 = 模块数 / 原天数
   - 新新鲜度指数 = 模块数 / 新天数
   - 评估新鲜度变化对玩家疲劳的影响
   - 如需维持新鲜度，计算需要补充的模块数量

3. **玩家疲劳曲线预测**
   - 基于天数延长/缩短，预测疲劳累积曲线变化
   - 识别新的流失拐点（通常7天、14天、21天为关键节点）
   - 评估长周期下的"中段倦怠"风险

4. **付费节奏影响**
   - 付费窗口延长/缩短对收入曲线的影响
   - 累充档位达成率预测（周期延长→达成更容易，但紧迫感降低）
   - 战令/通行证类周期绑定活动的连带影响

5. **留存拐点分析**
   - 预测调整后的次日/7日/14日留存变化
   - 识别关键留存干预节点（如第7天若缺乏内容，留存可能骤降）

6. **调整建议**
   - 模块增减建议（维持新鲜度指数 >= 0.5）
   - 峰值重排建议（主/次/收尾峰值间隔重新设计）
   - 里程碑调整建议（进度里程碑天数适配新周期）
   - 风险预警（高风险项标注红色）
