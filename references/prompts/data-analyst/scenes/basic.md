# Scene: DAT-basic - 基础数据分析

## 场景标识
- **ID**: DAT-basic
- **名称**: 基础数据分析
- **岗位**: 数据分析师
- **优先级**: P0（Phase 1）
- **适用场景**: 当你需要基于原始游戏数据进行日常运营监控、版本效果评估、玩家行为洞察等通用分析时。

## 场景目的
用户提供原始游戏数据（表格、CSV、文本描述等形式），明确分析维度和核心目标，输出结构化数据分析报告。适用于日常运营监控、版本效果评估、玩家行为洞察等通用场景。

## 处理逻辑

### 核心原则：工具优先，杜绝幻觉

**严禁**直接基于大模型记忆或推理输出数据结论。所有数值计算、统计汇总、趋势判断必须**先通过工具执行**，再将工具输出作为分析依据。

> ❌ 错误做法：用户给出数据表格后，模型直接心算均值/增长率并给出结论。
> ✅ 正确做法：用户给出数据后，模型调用 xlsx skill 或执行 Python 脚本计算，基于工具返回的精确结果输出结论。

### 工具调用（前置步骤，必须执行）

**第一步：判断数据形式并加载**

1. **文件数据处理（xlsx skill）—— 优先使用**
   - 当用户上传 `.xlsx` 或 `.csv` 文件时，**必须**调用 xlsx skill 读取数据
   - 使用 xlsx skill 执行：数据清洗、透视表汇总、基础统计计算、简单图表生成
   - 将工具输出的统计结果和图表作为**唯一可信**的分析依据
   - **禁止**在 xlsx skill 返回结果前，对数据做任何数值判断

2. **脚本计算（execute_command）—— 复杂计算必须**
   - 当需要以下任何计算时，**必须**调用 execute_command 执行 Python/R 脚本：
     - 描述性统计（均值、中位数、标准差、分位数等）
     - 增长率、环比、同比、变化幅度
     - 回归分析、聚类、时间序列分解、假设检验
     - 任意涉及多行数据运算的场景
   - 脚本执行后，将输出结果整合到分析报告中
   - 所有脚本执行前需向用户说明计算目的，执行后**展示原始输出**（截图或粘贴关键结果）
   - **禁止**基于脚本输出做二次心算修正

3. **外部数据检索（web_search）—— 辅助参考**
   - 当分析需要行业基准数据、竞品公开数据、或宏观背景信息时，调用 web_search 检索
   - 检索结果仅作为参考对比，需在报告中标注数据来源和时效性
   - 外部数据**不得**用于修正或替代用户提供的原始数据

### 数据可信度分层

| 数据来源 | 可信度 | 使用规则 |
|---|---|---|
| 工具执行结果（xlsx/Python） | ⭐⭐⭐⭐⭐ 最高 | 可直接用于结论 |
| 用户提供的原始数据 | ⭐⭐⭐⭐ 高 | 需经工具验证后使用 |
| 外部检索数据 | ⭐⭐⭐ 中 | 仅作参考，需标注来源 |
| 模型推理/心算 | ⭐ 极低（禁止） | **严禁**作为结论依据 |

### 分析流程

1. **数据质量检查**
   - 检查数据完整性（缺失值比例、缺失模式）
   - 检查异常值（明显超出合理范围的数值）
   - 检查数据一致性（同一指标口径是否统一）
   - 如有数据质量问题，在报告中明确标注并说明处理方式

2. **描述性统计**
   - 输出各关键指标的统计概览（计数、均值、中位数、标准差、最小值、最大值）
   - 识别数据分布特征（正态/偏态/长尾）

3. **按分析维度展开分析**
   - 时间维度：趋势变化、周期性、拐点识别
   - 分层维度：不同群体的差异对比（如付费分层、活跃分层、新老用户）
   - 交叉维度：多维度交叉分析（如时间×用户分层）

4. **异常检测与归因**
   - 识别数据中的异常点或异常模式
   - 结合业务背景进行归因分析，区分相关性与因果性
   - 对无法归因的异常，明确标注"待进一步验证"

5. **结论与建议**
   - 基于数据给出明确的分析结论
   - 提供可落地的业务建议，标注优先级（P0/P1/P2）
   - 给出预期效果（尽量量化）

## 质量自检

### 工具使用检查（最高优先级）
- [ ] **所有数值结论均来自工具执行结果**，未进行任何心算或推理估算
- [ ] 使用了 xlsx skill 或 Python 脚本处理原始数据，**不是**直接读取表格后口头计算
- [ ] 复杂统计（均值/增长率/回归/聚类等）**必须**有脚本执行记录或 xlsx skill 输出佐证
- [ ] 如果未使用工具（如数据量极小），需在报告中明确标注"本次分析基于直观判断，未使用计算工具"

### 分析质量检查
- [ ] 所有结论都有具体数据支撑，未出现"感觉""可能"等模糊表述
- [ ] 异常检测部分标注了置信度
- [ ] 建议部分标注了优先级和预期效果
- [ ] 明确说明了数据质量问题和处理方式
- [ ] 分析局限性已诚实披露
