# Scene: SYS-ux - 系统用户体验评估

## 场景标识
- **ID**: SYS-ux
- **名称**: 系统用户体验评估
- **岗位**: 系统策划
- **优先级**: P0（Phase 2）
- **适用场景**: 当一套系统方案已制定完成，需要评估用户使用该系统时的感受、认知负荷、心流体验和操作效率时。

## 场景目的

输入系统PRD/规则文档和功能流程说明，从用户视角出发，绘制用户旅程地图，评估每步的认知负荷和心流状态，识别挫败感节点和效率瓶颈，输出系统体验优化建议报告。

## 处理逻辑

1. **用户旅程地图绘制**
   - 基于输入的操作流程，绘制完整的用户旅程
   - 标注每个步骤的：用户目标、用户行为、系统反馈、情感状态
   - 识别旅程阶段：触发→进入→操作→反馈→退出

2. **认知负荷评估**
   - 对每个步骤评估信息密度（1-5分）
   - 评估决策复杂度（1-5分）
   - 评估记忆负担（是否需要记住前序步骤信息）
   - 绘制认知负荷热力图

3. **心流通道分析**
   - 基于心流理论（技能与挑战平衡），评估每个步骤是否处于心流通道
   - 识别"焦虑区"（挑战过高）：复杂规则、多选决策、高门槛操作
   - 识别"无聊区"（挑战过低）：重复点击、冗长等待、无意义确认
   - 绘制心流通道散点图

4. **反馈及时性检查**
   - 检查每个操作后是否有明确的系统反馈
   - 评估反馈的清晰度（玩家是否知道操作成功了/失败了/进行中了）
   - 识别"反馈缺失"节点（操作后玩家不知道发生了什么）

5. **挫败感识别**
   - 冗余操作：需要多次点击/跳转才能完成一个简单目标
   - 死胡同：用户操作后无法找到下一步的明确路径
   - 信息缺失：关键信息隐藏过深或展示时机不当
   - 预期违背：系统反馈与玩家心理预期不一致

6. **操作效率优化**
   - 计算完成核心目标的最短操作路径
   - 识别可合并/删减的步骤
   - 建议快捷方式、批量操作、智能默认值
   - 对比优化前后的操作步数

7. **体验优化建议**
   - P0：必须修复的挫败感问题（死胡同、反馈缺失、严重认知过载）
   - P1：建议优化项（冗余操作、路径过长、信息层级混乱）
   - P2：体验提升项（动效优化、情感化设计、个性化推荐）
