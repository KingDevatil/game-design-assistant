# Scene: SYS-prd - GDD/PRD文档生成

## 场景标识
- **ID**: SYS-prd
- **名称**: GDD/PRD文档生成
- **岗位**: 系统策划
- **优先级**: P0（Phase 1）

## 场景目的
输入需求分析文档（SYS-requirement输出），生成符合团队规范、可直接进入评审的GDD/PRD初稿。文档需包含：系统概述、详细规则、界面原型、数据结构设计、异常处理、测试要点。

## 输入模板

```markdown
## PRD生成输入

### 前置输出（请粘贴SYS-requirement的结果）
- [ ] 需求分析文档（SYS-requirement输出）

### 文档类型选择
- [ ] GDD（游戏设计文档，偏概念和规则）
- [ ] PRD（产品需求文档，偏功能和实现）
- [ ] 两者结合

### 团队规范要求
- 文档模板版本：_______
- 必须包含的章节：_______
- 特殊格式要求：_______（如：必须有流程图、必须有数据表格）

### 补充信息
- 目标阅读者：_______（程序/美术/测试/运营）
- 预计页数/篇幅：_______
- 是否有参考文档：_______
```

## 处理逻辑

1. **文档结构设计**
   - 根据文档类型（GDD/PRD）和团队规范，确定章节结构
   - 确保覆盖：概述、规则、流程、界面、数据、异常、测试

2. **系统概述撰写**
   - 一句话描述系统核心玩法
   - 系统定位（解决什么问题、服务什么目标）
   - 系统与核心循环的关系

3. **详细规则设计**
   - 核心玩法规则（主流程）
   - 子系统规则（如商店规则、排行榜规则）
   - 边界规则（次数限制、时间限制、等级限制）

4. **流程图设计**
   - 主流程（玩家从入口到出口）
   - 异常流程（错误提示、重试机制）
   - 状态机（如有复杂状态切换）

5. **界面原型描述**
   - 每个界面的布局描述（用文字描述，标注关键元素位置）
   - 交互逻辑（点击、滑动、拖拽的响应）
   - 状态变化（空态、加载态、错误态）

6. **数据结构设计**
   - 关键数据表/配置表结构
   - 字段定义、数据类型、取值范围
   - 关联关系

7. **异常与边界处理**
   - 网络异常、数据异常、操作异常的处理方案
   - 并发场景处理

8. **测试要点**
   - 功能测试用例要点
   - 性能测试关注点
   - 兼容性测试关注点
