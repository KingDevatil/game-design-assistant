# Scene: SYS-goal - 设计目的明确与梳理

## 场景标识
- **ID**: SYS-goal
- **名称**: 设计目的明确与梳理
- **岗位**: 系统策划
- **优先级**: P0（Phase 1）
- **适用场景**: 当你有一个模糊的系统设计需求，需要梳理核心目标、约束条件和成功指标时。

## 场景目的
输入模糊的初始需求描述，帮助策划梳理核心目标、约束条件、成功指标、设计边界，并提出待澄清问题。

## 输入模板

```markdown
## 初始需求描述
{{user_requirement}}

## 项目背景（选填）
- 游戏类型：{{game_type | default("MMORPG")}}
- 题材风格：{{theme | default("奇幻")}}
- 目标平台：{{platform | default("PC+移动端")}}
- 当前版本阶段：{{version_stage | default("正式上线")}}

## 约束条件（选填）
- 开发周期：{{dev_cycle}}
- 技术约束：{{tech_constraints}}
- 美术资源限制：{{art_limitations}}
- 预算范围：{{budget}}

## 目标用户画像（选填）
- 核心用户年龄段：{{user_age}}
- 用户付费意愿：{{user_willingness_to_pay}}
- 用户游戏经验：{{user_experience}}
```

## 处理逻辑
1. 分析需求描述中的关键词，识别核心诉求
2. 结合游戏类型特征，推断隐含目标
3. 基于项目约束，划定设计边界
4. 提出至少5个待澄清问题
5. 按优先级排序输出内容

## 质量自检
输出完成后，请自检以下项目：
- [ ] 待澄清问题数量 ≥ 5个
- [ ] 核心目标不超过2句话

- [ ] 设计边界的"排除范围"已明确列出
- [ ] 所有建议标注了优先级（P0/P1/P2）
