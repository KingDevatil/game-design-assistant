# Scene: SYS-uiux - UI/UX交互方案设计

## 场景标识
- **ID**: SYS-uiux
- **名称**: UI/UX交互方案设计
- **岗位**: 系统策划
- **优先级**: P1（Phase 2）

## 场景目的
输入系统功能需求和PRD文档，输出该系统的UI界面布局方案、交互流程、状态变化和视觉规范建议，帮助美术和程序理解界面需求。

## 输入模板

```markdown
## UI/UX设计输入

### 系统功能需求
请粘贴系统的核心功能列表：
_______

### 已有约束
- 屏幕分辨率/安全区：_______
- UI风格规范：_______（如：已有中国风、科幻风）
- 交互范式：_______（如：点击为主、支持滑动、不支持拖拽）
- 性能约束：_______（如：单个界面drawcall不超过50）

### 目标平台
- [ ] 手机（竖屏）
- [ ] 手机（横屏）
- [ ] 平板
- [ ] PC

### 参考竞品界面
- 竞品A界面截图/描述：_______
- 竞品B界面截图/描述：_______
```

## 处理逻辑

1. **信息架构梳理**
   - 列出系统涉及的所有信息单元
   - 按重要性分层：一级信息（必须看到）、二级信息（经常查看）、三级信息（偶尔查看）

2. **界面结构规划**
   - 确定需要几个界面/页面
   - 确定界面间的跳转关系
   - 确定每个界面的核心功能

3. **布局设计**
   - 基于平台特性（手机竖屏/横屏）设计布局
   - 标注关键元素的位置和尺寸比例
   - 考虑拇指热区和操作便利性

4. **交互流程设计**
   - 主操作流程的交互步骤
   - 快捷操作和批量操作
   - 反馈机制（点击反馈、状态变化、动画提示）

5. **状态设计**
   - 空态（无数据时的展示）
   - 加载态
   - 错误态
   - 完成态

6. **视觉规范建议**
   - 配色建议（基于项目主题色）
   - 字体层级
   - 图标风格
   - 间距规范
