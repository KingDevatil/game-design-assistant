# Scene: CBT-skill - 技能/连招设计

## 场景标识
- **ID**: CBT-skill
- **名称**: 技能/连招设计
- **岗位**: 战斗策划
- **优先级**: P1（Phase 2）

## 场景目的
输入角色的定位和战斗风格，设计完整的技能组和连招体系，确保操作深度、连招流畅性和爽点节奏。

```markdown

- 技能数量限制：_______

- 必须体现的题材典故：_______

### 参考连招
- 竞品参考：_______
- 期望连招长度：_______
- 期望连招爽点：_______
```

## 处理逻辑

1. **技能框架设计**
   - 确定技能间的功能分工（起手/衔接/终结/控制/位移）
   - 确保技能组覆盖不同战斗场景（单体/群攻/控制/生存）

2. **连招路径设计**
   - 设计基础连招（简单易学）
   - 设计进阶连招（高伤害但操作要求高）
   - 设计极限连招（秀操作，容错低）

3. **取消链设计**
   - 哪些技能可以互相取消
   - 取消窗口的帧数设计

4. **爽点节奏**
   - 连招中的伤害爆发点分布
   - 奥义终结的仪式感设计

5. **题材元素融合**
   - 技能名与题材典故/世界观呼应
   - 奥义演出与题材名场面结合
