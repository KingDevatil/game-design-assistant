# 自定义场景索引

> 用户可在项目知识包中定义**专属场景**，扩展 Skill 的能力覆盖。
> 自定义场景遵循与内置场景相同的结构规范，优先级高于同ID的内置场景。

## 按角色分类

自定义场景按角色ID分目录存放，每个角色目录下有自己的 `INDEX.md`：

```
scenes/
├── INDEX.md                    # 本文件：全局索引
├── system-designer/            # 系统策划自定义场景
│   ├── INDEX.md                #   该角色的场景索引
│   └── {scene_id}.md           #   场景文件
├── writer/                     # 文案策划自定义场景
│   ├── INDEX.md
│   └── ...
├── balance-designer/           # 数值策划
├── level-designer/             # 关卡策划
├── combat-designer/            # 战斗策划
├── event-designer/             # 活动策划
└── data-analyst/               # 数据分析师
```

## 使用规则

1. **按角色分类**：每个角色一个子目录，目录名与角色ID一致
2. **角色级索引**：每个子目录下的 `INDEX.md` 声明该角色的自定义场景
3. **角色子目录索引格式**：

   | 场景ID | 名称 | 触发关键词 | 描述 |
   |--------|------|-----------|------|
   | `{id}` | {名称} | {关键词1}、{关键词2} | {描述} |

   > 不再需要在索引中填写"关联角色"，因为目录名已标识角色。
4. **文件命名**：场景文件名建议与场景ID一致（如 `boss_design.md`）
5. **场景ID**：建议使用有意义的英文标识（如 `CBT-boss_design`），避免与内置场景ID混淆
6. **优先级**：自定义场景 > 内置场景 — 同ID时以自定义场景为准（P0 级覆盖）
