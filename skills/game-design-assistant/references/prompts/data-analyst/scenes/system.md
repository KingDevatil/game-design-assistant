# Scene: DAT-system - 系统专项数据分析

## 场景标识
- **ID**: DAT-system
- **名称**: 系统专项数据分析
- **岗位**: 数据分析师
- **优先级**: P0（Phase 1）
- **适用场景**: 当你需要对特定游戏系统进行深度数据分析（经济、战斗、关卡、PVP、社交、运营活动等）时。

## 场景目的
针对特定游戏系统（经济、战斗、关卡、PVP、社交、运营活动等）进行深度数据分析。需要结合系统设计规则（`systems/` 知识库）和项目专属知识（由 `user_state.yaml` 中 `knowledge_source` 配置的数据源），输出系统调优建议。适用于系统设计验证、上线后效果复盘、系统性问题诊断。

## 处理逻辑

### 核心原则：工具优先，杜绝幻觉

**严禁**直接基于大模型记忆或推理输出数据结论。所有数值计算、统计汇总、趋势判断必须**先通过工具执行**，再将工具输出作为分析依据。

> ❌ 错误做法：用户给出数据表格后，模型直接心算均值/增长率并给出结论。
> ✅ 正确做法：用户给出数据后，模型调用 xlsx skill 或执行 Python 脚本计算，基于工具返回的精确结果输出结论。

### 工具调用（前置步骤，必须执行）

**第一步：判断数据形式并加载**

1. **文件数据处理（xlsx skill）—— 优先使用**
   - 当用户上传 `.xlsx` 或 `.csv` 文件时，**必须**调用 xlsx skill 读取数据
   - 使用 xlsx skill 执行：数据清洗、透视表汇总、基础统计计算、简单图表生成
   - 将工具输出的统计结果和图表作为**唯一可信**的分析依据
   - **禁止**在 xlsx skill 返回结果前，对数据做任何数值判断

2. **脚本计算（execute_command）—— 复杂计算必须**
   - 当需要以下任何计算时，**必须**调用 execute_command 执行 Python/R 脚本：
     - 描述性统计（均值、中位数、标准差、分位数等）
     - 增长率、环比、同比、变化幅度
     - 系统耦合度计算、经济模型模拟、战斗平衡模拟
     - 关卡通过率建模、回归分析、聚类、时间序列分解
     - 任意涉及多行数据运算的场景
   - 脚本执行后，将输出结果整合到分析报告中
   - 所有脚本执行前需向用户说明计算目的，执行后**展示原始输出**（截图或粘贴关键结果）
   - **禁止**基于脚本输出做二次心算修正

3. **外部数据检索（web_search）—— 辅助参考**
   - 当分析需要行业基准数据、同类系统的公开数据、或竞品系统参数时，调用 web_search 检索
   - 检索结果仅作为参考对比，需在报告中标注数据来源和时效性
   - 外部数据**不得**用于修正或替代用户提供的原始数据

### 数据可信度分层

| 数据来源 | 可信度 | 使用规则 |
|---|---|---|
| 工具执行结果（xlsx/Python） | ⭐⭐⭐⭐⭐ 最高 | 可直接用于结论 |
| 用户提供的原始数据 | ⭐⭐⭐⭐ 高 | 需经工具验证后使用 |
| 外部检索数据 | ⭐⭐⭐ 中 | 仅作参考，需标注来源 |
| 模型推理/心算 | ⭐ 极低（禁止） | **严禁**作为结论依据 |

### 分析流程

1. **系统规则对齐**
   - 加载 `references/game_context/systems/` 中对应系统的通用设计规则
   - 根据 `knowledge_source` 配置加载项目专属知识中的项目专属规则
   - 明确系统的核心设计目标和关键约束

2. **系统健康度评估**
   - 对照系统设计目标，评估当前数据是否达标
   - 识别与预期偏离的指标，量化偏离程度

3. **多维度诊断**
   - 玩家分层维度：不同付费/活跃/等级层级的表现差异
   - 时间维度：系统数据随时间的变化趋势
   - 关联系统维度：与其他系统的耦合影响（如经济系统对留存的影响）

4. **根因分析**
   - 对发现的问题进行5Why式归因
   - 区分设计问题、配置问题、玩家行为变化、外部因素

5. **调优方案**
   - 提供2-3组备选调优方案
   - 每组方案包含：具体调整项、预期效果、实施难度
   - 推荐首选方案并说明理由

## 质量自检

### 工具使用检查（最高优先级）
- [ ] **所有数值结论均来自工具执行结果**，未进行任何心算或推理估算
- [ ] 使用了 xlsx skill 或 Python 脚本处理原始数据，**不是**直接读取表格后口头计算
- [ ] 复杂统计（均值/增长率/回归/聚类等）**必须**有脚本执行记录或 xlsx skill 输出佐证
- [ ] 如果未使用工具（如数据量极小），需在报告中明确标注"本次分析基于直观判断，未使用计算工具"

### 分析质量检查
- [ ] 分析结合了系统设计规则（通用+项目专属）
- [ ] 问题归因链完整，至少进行了2层归因
- [ ] 提供了2组以上备选调优方案
- [ ] 所有调优建议标注了预期效果和验证指标
- [ ] 区分了设计问题、配置问题、玩家行为变化、外部因素
