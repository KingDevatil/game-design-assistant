# Scene: CBT-frame - 帧数据分析

## 场景标识
- **ID**: CBT-frame
- **名称**: 帧数据分析
- **岗位**: 战斗策划
- **优先级**: P1（Phase 2）

## 场景目的
输入技能的帧数据表（启动帧、 active帧、 恢复帧、取消窗口等），分析技能性能，识别强度问题（如过长的恢复帧、过宽的判定框），输出调优建议。

```markdown

### 分析维度
- [ ] 技能性能对比（同类技能的帧数差异）
- [ ] 强度评估（帧数优势/劣势）
- [ ] 连招可行性（取消窗口是否足够）
- [ ] 手感问题（前后摇是否流畅）

### 参考标准
- 同类型技能平均启动帧：_______
- 可接受的取消窗口下限：_______帧
- 技能性能差异容忍度：±_______%
```

## 处理逻辑

1. **帧数解析**
   - 计算每个技能的总帧数、有效帧占比
   - 计算技能间的帧数优势差

2. **性能评估**
   - 启动帧：越短越优（先手优势）
   - Active帧：影响判定持续时间和命中窗口
   - 恢复帧：越短越安全
   - 取消窗口：影响连招深度

3. **异常检测**
   - 启动帧异常长（难以命中）
   - 恢复帧异常长（容易被惩罚）
   - 取消窗口过小（连招断裂）
   - 性能与伤害不匹配（高伤害+低风险 = 过强）

4. **调优建议**
   - 对每个异常项给出帧数调整建议
