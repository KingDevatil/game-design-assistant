# Scene: CBT-unit - 战斗单位设计

## 场景标识
- **ID**: CBT-unit
- **名称**: 战斗单位设计
- **岗位**: 战斗策划
- **优先级**: P0（Phase 2）

## 场景目的
输入角色/敌人/宠物的定位需求和设计约束，输出完整的战斗单位方案，包括：基础属性框架、技能组设计、操作手感参数、成长曲线。

## 输入模板

```markdown
## 战斗单位设计输入

### 基础信息
- 单位名称：_______
- 单位类型：_______（玩家角色/Boss/精英/小怪/宠物）
- 定位标签：_______（如：突进型、力量型、均衡型）
- 参考角色：_______（如：参考某已知角色的灵活突进）

### 设计约束
- 星级/稀有度：_______
- 可操作复杂度：_______（简单/中等/复杂）
- 目标玩家群体：_______
- 是否需要与某角色形成克制关系：_______

### 题材适配
- 题材原型：_______（如：历史人物、神话角色、原创设定）
- 武器类型：_______
- 标志性特征：_______
```

## 处理逻辑

1. **定位分析**
   - 基于定位标签确定属性倾向（攻/防/血/速的优先级）
   - 确定操作复杂度对应的连招深度

2. **基础属性框架**
   - 设计1级和满级的基础属性值
   - 确保与同定位角色的属性有差异化

3. **技能组设计**
   - 普攻：段数、范围、手感
   - 技能1-3：效果、CD、消耗、连招衔接
   - 奥义/必杀：伤害、演出、触发条件

4. **手感参数**
   - 前后摇帧数、取消窗口、硬直时间
   - 移动速度、攻击范围、受击反馈

5. **成长曲线**
   - 属性成长系数
   - 技能解锁/升级节点
