# Scene: NUM-dda - DDA动态难度调节

## 场景标识
- **ID**: NUM-dda
- **名称**: DDA动态难度调节
- **岗位**: 数值策划
- **优先级**: P1（Phase 2）

## 场景目的
输入玩家行为数据（胜率、死亡次数、通关时间等），设计动态难度调整方案，使不同水平玩家都能获得流畅体验（既不过难也不过易）。

## 处理逻辑

1. **玩家分层建模**
   - 基于胜率、死亡次数、通关时间将玩家分为 struggling/flow/bored 三层
   - 计算各层占比，识别当前体验问题最突出的群体

2. **调整维度设计**
   - 从输入中选择 2-3 个核心调整维度（优先选不可被玩家感知的维度）
   - 为每个维度设计分层系数表（基准 100%，struggling 下调，bored 上调）

3. **状态转换规则**
   - 定义升级/降级触发条件（连续失败次数、胜率阈值、评估周期）
   - 设计回退机制，防止难度无限下降或无限上升

4. **生命周期适配**
   - 新手期（1-3天）：偏宽松，建立信心
   - 成长期（4-14天）：逐步提升挑战，暴露真实水平
   - 成熟期（15天+）：根据真实水平稳定，允许小幅波动

5. **隐形性校验**
   - 检查所有调整是否可被玩家明显感知
   - 对感知风险高的调整，改用"辅助"包装（如暴击率补偿→"幸运加成"）

6. **效果评估设计**
   - 定义关键指标：平均胜率、3连败率、DDA感知率
   - 设定评估周期和预警阈值

| 掉落提升 | "首通奖励" | 正常机制 |

### 6.2 避免感知的设计原则
1. 不使用明显的数值显示
2. 变化幅度控制在±20%以内
3. 使用随机波动掩盖调整
4. 与玩家行为挂钩（如"连击奖励"）
