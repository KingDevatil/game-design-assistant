# Scene: NUM-anomaly - 数据异常检测

## 场景标识
- **ID**: NUM-anomaly
- **名称**: 数据异常检测
- **岗位**: 数值策划
- **优先级**: P0（Phase 2）

## 场景目的
输入数值配置表，自动识别配置中的异常模式：如数值跳变、重复配置、逻辑矛盾、越界值等，输出异常清单和修复建议。

## 输入模板

```markdown
## 数据异常检测输入

### 待检测数据表
请粘贴CSV或表格数据：
| ID | ... |
|---|---|

### 检测规则（可选，未提供则使用默认规则）
- [ ] 数值跳变检测（相邻等级/品质间增幅异常）
- [ ] 重复配置检测（相同效果不同ID）
- [ ] 逻辑矛盾检测（如：需求等级 > 使用等级）
- [ ] 越界值检测（超出预设范围）
- [ ] 空值/缺失检测
- [ ] 公式依赖检测（如：总属性 ≠ 子属性之和）
```

## 处理逻辑

1. **格式校验**
   - 检查数据类型是否匹配字段定义
   - 检查必填字段是否有空值

2. **数值跳变检测**
   - 计算相邻等级/品质间的增幅百分比
   - 识别增幅超过预设阈值（如±50%）的跳变

3. **重复配置检测**
   - 对比不同ID的配置，识别效果完全相同的重复项
   - 识别相似度超过90%的近重复项

4. **逻辑矛盾检测**
   - 检查字段间的逻辑关系（如：最小值 ≤ 最大值）
   - 检查跨表引用的一致性

5. **越界检测**
   - 检查数值是否超出合理范围（如：概率 > 100%）
   - 检查字符串长度是否超限

6. **输出修复建议**
   - 对每个异常给出修复建议
   - 标注异常的严重程度
