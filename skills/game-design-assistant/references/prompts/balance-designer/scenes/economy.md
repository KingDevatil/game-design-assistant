# Scene: NUM-economy - 经济系统模拟

## 场景标识
- **ID**: NUM-economy
- **名称**: 经济系统模拟
- **岗位**: 数值策划
- **优先级**: P1（Phase 2）

## 场景目的
输入经济系统配置（产出/消耗/汇率/回收），模拟玩家在游戏生命周期内的经济走势，预警通胀/通缩风险，评估付费设计合理性。

## 处理逻辑

1. **模型初始化**
   - 根据玩家分层初始化各群体的经济行为参数
   - 设定产出/消耗的基础概率分布

2. **日循环模拟**
   - 逐日模拟每个玩家的：收入 → 消费 → 储蓄 → 投资
   - 考虑随机波动（如暴击产出、活动加成）

3. **关键指标计算**
   - 货币总量变化曲线
   - 人均持有量变化
   - 贫富差距（基尼系数）
   - 付费转化率/ARPPU

4. **风险识别**
   - 通胀：货币总量增速 > 商品/服务增速
   - 通缩：货币被过度回收，流通不足
   - 贫富差距：头部玩家垄断资源

5. **调优建议**
   - 产出/消耗调整建议
   - 回收机制优化
   - 付费点调整
