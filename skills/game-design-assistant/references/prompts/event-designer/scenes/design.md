# Scene: EVT-design - 活动需求分析

## 场景标识
- **ID**: EVT-design
- **名称**: 活动需求分析
- **岗位**: 活动策划
- **优先级**: P0（Phase 2）
- **适用场景**: 当你已明确活动目标和竞品参考，需要细化活动规则、奖励结构、阶段节奏和UI需求时。

## 场景目的
输入活动设计目的（EVT-goal）和竞品调研（EVT-research），结合项目现状，输出活动需求分析文档，明确活动规则、奖励、节奏、UI的具体需求。

```markdown

- [ ] 竞品调研报告（EVT-research）

- 可复用的活动模板：_______
- 美术资源可用情况：_______

### 需求范围
- [ ] 活动规则详细设计
- [ ] 奖励结构设计
- [ ] 阶段节奏设计
- [ ] UI/交互需求
```

## 处理逻辑

1. **规则设计**
   - 基于竞品借鉴和项目约束，设计活动核心规则
   - 确保规则简单易懂，降低理解成本

2. **奖励结构设计**
   - 设计分层奖励（免费/小R/中R/大R）
   - 确保各层级奖励有吸引力且成本可控

3. **节奏设计**
   - 确定活动的阶段和每个阶段的核心玩法
   - 设计阶段间的递进关系和奖励递增

4. **技术可行性评估**
   - 评估规则是否在当前技术框架内可实现
   - 识别需要新增开发的内容

5. **数据监控设计**
   - 确定需要服务端记录的关键日志（参与、奖励发放、异常）
   - 不需要客户端逐点击埋点
