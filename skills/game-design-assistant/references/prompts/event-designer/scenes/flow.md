# Scene: EVT-flow - 活动心流体验分析

## 场景标识
- **ID**: EVT-flow
- **名称**: 活动心流体验分析
- **岗位**: 活动策划
- **优先级**: P0（Phase 2）
- **适用场景**: 当你已有活动方案（含模块清单和时序规划），需要评估玩家体验曲线、诊断心流失衡风险时

## 场景目的

输入现有活动方案（模块清单+时序+峰值锚点），输出心流体验分析报告，识别心流失衡风险，并给出多峰结构设计建议。


## 处理逻辑

1. **心流理论框架引入**
   - 引用 Mihaly Csikszentmihalyi 心流理论八要素
   - 引用活动节奏曲线设计方法论（单峰/多峰/阶梯/爆发型）
   - 引用内容新鲜度模型：新鲜度指数 = 模块数量 / 活动天数

2. **模块心流价值分析**
   - 基于输入信息，为每个模块评估心流贡献值（1-5分）
   - 标注每个模块的参与门槛和付费关联
   - 按时间轴绘制模块时序分布图（ASCII）

3. **周期节奏诊断**
   - 计算整体内容新鲜度指数
   - 按阶段（预热/正式/收尾）计算新鲜度
   - 识别心流失衡风险节点：空窗期、断崖期、阻塞点
   - 绘制心流强度曲线（ASCII 图表）

4. **多峰结构设计**
   - 识别当前方案的主峰值（不可替代锚点）
   - 评估是否需要次峰值（总天数>=14天建议至少2个）
   - 评估收尾峰值设计（活动结束前3-5天）
   - 输出优化后的多峰结构建议

5. **执行优先级建议**
   - P0：必须修复的心流失衡问题（如严重空窗期）
   - P1：建议优化项（如峰值间隔调整）
   - P2：锦上添花项（如里程碑包装优化）
   - 每项标注改动量和预期收益

6. **待确认问题输出**
   - 至少列出3个需要与团队确认的问题
   - 标注阻塞级别（阻塞/重要/建议）
