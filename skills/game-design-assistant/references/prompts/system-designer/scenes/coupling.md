# Scene: SYS-coupling - 系统耦合分析

## 场景标识
- **ID**: SYS-coupling
- **名称**: 系统耦合分析
- **岗位**: 系统策划
- **优先级**: P1（Phase 2）

## 场景目的
输入一个或多个系统设计方案，分析系统间的耦合关系、数据流向、潜在冲突，输出系统耦合关系图和优化建议，降低系统间依赖风险。

```markdown
- [ ] 循环依赖检测
- [ ] 性能影响评估
- [ ] 扩展性评估
```

## 处理逻辑

1. **系统清单梳理**
   - 列出所有涉及的系统及其核心功能
   - 标注每个系统的数据输入和输出

2. **耦合关系图构建**
   - 识别直接依赖（A调用B）
   - 识别间接依赖（A通过C影响B）
   - 识别数据共享（A和B读写同一份数据）

3. **冲突检测**
   - 功能冲突：两个系统对同一功能有不同定义
   - 数值冲突：两个系统的数值设计互相冲击
   - 逻辑冲突：规则之间存在矛盾

4. **风险评级**
   - 高耦合：修改A必然影响B，风险高
   - 中耦合：修改A可能影响B，需关注
   - 低耦合：修改A基本不影响B，理想状态

5. **解耦建议**
   - 对高耦合关系提出解耦方案
   - 建议中间层/抽象层设计
   - 建议数据隔离方案
