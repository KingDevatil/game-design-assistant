# Scene: SYS-exception - 异常场景枚举

## 场景标识
- **ID**: SYS-exception
- **名称**: 异常场景枚举
- **岗位**: 系统策划
- **优先级**: P1（Phase 2）

## 场景目的
输入系统PRD/规则文档，穷举该系统在运行过程中可能遇到的所有异常场景、边界条件和 corner case，输出异常场景清单及处理方案，帮助程序和测试完善异常处理。

```markdown

请粘贴系统的PRD或核心规则描述：
_______

- 涉及的玩家数据：_______
- 涉及的配置数据：_______
- 涉及的外部系统：_______

### 枚举范围
- [ ] 玩家操作异常
- [ ] 网络/服务器异常
- [ ] 数据异常（越界/空值/并发）
- [ ] 时间/定时器异常
- [ ] 跨系统交互异常
- [ ] 极端数值场景
```

## 处理逻辑

1. **正常流程拆解**
   - 将系统的主流程拆分为最小步骤单元
   - 每个步骤标注：输入、处理、输出

2. **单点异常注入**
   - 对每个步骤，假设输入异常（空值、越界、格式错误）
   - 假设处理异常（超时、中断、资源不足）
   - 假设输出异常（写入失败、通知丢失）

3. **组合异常场景**
   - 多个异常同时发生
   - 异常链式反应（A异常导致B异常）

4. **边界条件枚举**
   - 数值边界：最大值、最小值、零值、负值
   - 数量边界：空列表、单元素、满容量
   - 时间边界：跨天、跨月、闰年、时区切换
   - 状态边界：首次、末次、中间态

5. **分类与优先级**
   - 按影响程度分类：致命/严重/一般/提示
   - 按发生概率分类：高频/中频/低频
   - 按处理优先级排序

6. **处理方案建议**
   - 对每个异常场景给出建议的处理方式
   - 区分：阻断/降级/补偿/忽略
