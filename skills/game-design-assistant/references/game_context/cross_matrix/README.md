# 类型×题材交叉矩阵

交叉矩阵用于描述特定游戏类型与题材的融合设计原则，解决"同一类型在不同题材下如何差异化设计"的问题。

## 用途

当项目同时配置了 `project_type` 和 `project_theme`（见 `user_state.yaml`）时，Skill 会检查此目录下是否存在对应的交叉矩阵文件。若存在，作为 P2 层知识库加载，优先级高于单独的通用类型/题材文件（P4）。

## 命名格式

```
{type}_{theme}.md
```

- `{type}`：游戏类型 ID，与 `game_types/` 下的文件名对应（不含 `.md`）
- `{theme}`：游戏题材 ID，与 `themes/` 下的文件名对应（不含 `.md`）

## 内容规范

交叉矩阵文件应包含以下章节：

1. **核心融合原则** — 类型机制如何与题材气质结合
2. **角色/单位设计融合指南** — 题材元素如何融入角色设计
3. **关卡/场景设计融合指南** — 题材如何影响关卡结构和场景氛围
4. **文案风格融合指南** — 题材对文案基调和术语的影响
5. **商业化与运营融合指南** — 题材如何影响付费设计和活动主题

## 示例

假设项目配置为：

```yaml
project_type: "arpg"
project_theme: "sci_fi"
```

Skill 会尝试加载 `cross_matrix/arpg_sci_fi.md`。若文件存在，则作为 P2 层知识库注入。

> 注意：交叉矩阵是可选的。如果没有对应的交叉矩阵文件，Skill 将只加载独立的类型文件（P4）和题材文件（P4），不影响正常使用。
