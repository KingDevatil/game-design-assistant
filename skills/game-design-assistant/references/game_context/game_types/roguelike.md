# 游戏类型：Roguelike / Roguelite（肉鸽/轻度肉鸽）

> **对标参考**：《杀戮尖塔》、《哈迪斯》、《以撒的结合》、《死亡细胞》、《吸血鬼幸存者》、《土豆兄弟》  
> **核心标签**：随机生成、永久死亡、Build构筑、高难度、重复可玩性、每局不同

---

## 核心循环

```
开始一局 → 随机地图/敌人/道具 → 选择Build路线 → 战斗/探索 → 失败或通关 → 解锁新内容 → 再来一局
```

## Roguelike vs Roguelite 区别

| 维度 | Roguelike（传统） | Roguelite（轻度） |
|---|---|---|
| **死亡惩罚** | 永久死亡，一切归零 | 部分进度保留（解锁角色/道具/天赋） |
| **局外养成** | 无 | 有（Meta-progression） |
| **难度** | 极高 | 中高，可通过养成降低 |
| **受众** | 硬核玩家 | 更广泛 |
| **代表** | 《Rogue》、《Nethack》、《DCSS》 | 《哈迪斯》、《杀戮尖塔》、《以撒的结合》、《吸血鬼幸存者》 |

## 系统架构特征

| 系统模块 | 设计要点 |
|---|---|
| **随机生成** | 地图布局、敌人配置、道具掉落、事件选择，每局体验不同 |
| **道具Build** | 大量道具/卡牌/武器，随机获取，形成意想不到的组合（Synergy） |
| **局外解锁** | 通过游玩解锁新角色、新道具、新难度、剧情碎片 |
| **难度层级** | 多周目/多难度，每次增加新机制或更强敌人 |
| **局内成长** | 单局内通过道具/升级不断增强，形成正向反馈 |
| **付费设计** | 买断制为主，DLC扩展内容，少数F2P（局外加速、皮肤） |

## 核心设计要素

| 要素 | 说明 | 设计要点 |
|---|---|---|
| **随机性 vs 可控性** | 随机带来新鲜感，但不能完全看脸 | 设计保底机制、选择分支、策略空间 |
| **道具联动（Synergy）** | 道具A+道具B产生1+1>2的效果 | 设计足够多的联动组合，增加Build深度 |
| **风险与收益** | 高难度区域/事件有更好的奖励 | 玩家自主选择的博弈 |
| **局外进度** | 死亡后保留部分进度，降低挫败感 | 进度解锁要足够快，不能让玩家觉得白玩 |
| **操作简化** | 《吸血鬼幸存者》类自动攻击，降低操作门槛 | 重心放在Build选择和走位 |

## 关键设计约束

1. **挫败感管理**：永久死亡的惩罚要合理，失败原因要清晰（让玩家觉得"再来一局能赢"）
2. **Build多样性**：避免某几个Build过于强势，保持流派多样性
3. **随机公平性**：随机不是完全无序，要保证每局都有通关可能性
4. **内容量**：需要大量道具/敌人/事件支撑重复游玩，内容量是核心竞争力
5. **学习曲线**：早期容易上手，后期深度足够，难度曲线平滑

## 常见策划产出物
- 道具设计文档（效果、联动、稀有度）
- 敌人/Boss设计（攻击模式、阶段、克制关系）
- 地图生成规则（房间类型、连接规则、难度梯度）
- 事件设计（随机事件表、选择分支、风险收益）
- 解锁系统设计（解锁条件、进度曲线、奖励设计）
