# 游戏类型：MMORPG（大型多人在线角色扮演游戏）

> **对标参考**：《魔兽世界》、《最终幻想14》、《剑网3》、《逆水寒》  
> **核心标签**：大型开放世界、强社交、角色扮演、团队副本、公会战、经济系统

---

## 核心循环

```
创建角色 → 任务升级 → 装备获取 → 技能/天赋养成 → 社交组队 → 副本/PVP/生产 → 更高难度内容
```

## 系统架构特征

| 系统模块 | 设计要点 |
|---|---|
| **角色成长** | 等级 + 装备 + 技能树 + 天赋/符文/专精系统，多职业分工明确（坦克/治疗/DPS） |
| **战斗系统** | Tab-target或动作战斗，技能循环（Skill Rotation），团队配合机制（站位/打断/驱散） |
| **PVE内容** | 主线任务、支线任务、日常/周常、副本（普通/英雄/史诗）、团队副本（10-40人）、世界BOSS |
| **PVP内容** | 竞技场（2v2/3v3/5v5）、战场（大规模对抗）、公会战、野外PK、阵营对抗 |
| **经济系统** | 双货币（绑定+可交易）、拍卖行/交易行、生产采集（采矿/草药/锻造/附魔）、玩家交易 |
| **社交系统** | 公会/帮派、组队匹配、好友、师徒、婚姻、语音聊天、跨服社交 |
| **外观系统** | 时装/幻化、坐骑、宠物、翅膀、称号，不影响数值的外观付费 |
| **付费设计** | 月卡/季卡/点卡、外观时装、便利性道具、抽卡/宝箱、成长加速、增值服务 |

## 战斗系统详解

### 职业分工体系
| 职责 | 定位 | 核心任务 | 代表职业 |
|---|---|---|---|
| **坦克（Tank）** | 高生存、高仇恨 | 吸引Boss仇恨、承受伤害、保护队友 | 战士、骑士、守护者 |
| **治疗（Healer）** | 恢复、辅助 | 维持团队血量、驱散负面状态、增益队友 | 牧师、德鲁伊、吟游诗人 |
| **输出（DPS）** | 高伤害 | 最大化伤害输出、处理机制（转火/跑位） | 法师、盗贼、猎人、术士 |

### 副本机制设计
| 机制类型 | 说明 | 设计要点 |
|---|---|---|
| **站位机制** | Boss技能要求玩家分散/集合/排成直线 | 考验团队协调 |
| **转火机制** | 优先击杀召唤物/特定目标 | 考验DPS判断和切换目标能力 |
| **环境机制** | 场地变化（毒池、火焰、冰霜） | 考验走位和时机把握 |
| **阶段转换** | Boss血量到达阈值后改变技能和机制 | 增加战斗层次感和新鲜感 |
| **狂暴机制** | 战斗时间过长触发全屏秒杀 | 设定DPS检测门槛 |

## 关键设计约束

1. **免费玩家体验**：必须保证免费/低付费玩家有完整的PVE和社交体验，付费不能破坏核心玩法公平性
2. **社交是留存核心**：设计足够强的协作动机（团队副本奖励）和竞争动机（排行榜、公会排名）
3. **内容消耗速度**：MMO内容生产周期长，需设计可重复游玩内容（日常、周常、赛季、随机副本）
4. **经济系统健康**：防止通货膨胀、工作室刷金、物价崩溃，设计货币回收机制（修理费、拍卖行税）
5. **服务器技术**：大规模同屏、跨服匹配、分线/位面技术、延迟优化
6. **新手友好**：提供清晰的引导系统、导师系统、新手保护期，降低MMO的学习门槛
